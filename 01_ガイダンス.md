## 1. なぜシステム工学か

### 現代社会の3局面

| 局面 | 内容 | 例 |
|------|------|-----|
| システムの複雑化 | 部品・コード・ECUの爆発 | 航空機、自動車 |
| サービスの融合 | モノ→コト、Product Service System | カーシェア、サブスク |
| SoSの出現 | 複数システムの相互接続 | スマートシティ、サプライチェーン |

### System of Systems（SoS）の5特性

1. **運用上の独立性** — 各システムは単独で運用可能  
2. **管理上の独立性** — 別の管理主体  
3. **地理的分散**  
4. **創発的振る舞い** — 全体として新しい機能  
5. **進化的発展**

---

## 2. システムの定義

> 複数の構成要素が相互関連を持ち、全体として部分の総和以上の振る舞いを有する、目的を持った集合体（OR用語 Z8121 も参照）

| # | 特性 | 意味 |
|---|------|------|
| 1 | **複数要素** | 人・物・情報など非一様な要素の集合 |
| 2 | **相互作用** | 要素が互いに影響し合う |
| 3 | **目的性** | 全体が共通の目的に向かう |

### 概念図

- **環境** ↔ **境界** ↔ システム内部（要素A,B,C…）
- **INPUT**（物質・エネルギー・情報）→ **OUTPUT**
- 境界は「何をシステムに含めるか」の認識的選択（例：自転車に乗る人を含むか）

---

## 3. 境界の2つの見方

| | 存在論 | 認識論 |
|---|--------|--------|
| 立場 | 客観的に実在する区切り | 分析者がモデル化のために設定 |
| 含意 | 境界は固定 | 同じ対象でも観点で変わる |

---

## 4. 社会の5階層モデル（システム創成）

1. 自然法則・自然現象層  
2. 製造・生産・設計技術層  
3. 製品・作物・作品層  
4. サービス・システム運用層  
5. 制度・体制層  

各階層でアーキテクチャ設計し、階層間対応を明確化する。

---

## 5. システムの6つの認識側面

問題を多面的に捉える枠組み（試験では名称と要点）:

| 側面 | 問い |
|------|------|
| 技術的 | どう動くか |
| 人的 | 誰が関わるか |
| 組織的 | どの組織か |
| 経済的 | コスト・便益 |
| 政治的 | 権力・利害 |
| 倫理的 | 何が正しいか |

---

## 6. 問題の分類とアプローチ

| | 良定義問題 | 悪定義問題 |
|---|------------|------------|
| 目標 | 明確 | 曖昧・対立 |
| 関係者 | 合意しやすい | 価値観が割れる |
| 手法 | 最適化・シミュレーション | ソフトシステムズ、構造化（ISM等） |

**ソフトシステムズ方法論（Checkland）**: 悪定義問題に対し、関係者の世界観（Weltanschauung）を明らかにしながら学習するプロセス。

---

## 7. システムズアプローチのプロセス

問題把握 → 解決策の構想・評価 → 試行と評価（技術・経済・社会）→ 改善の反復。

**V-Model**: 要求 → 設計 → 実装 ↔ 検証・評価が対応。

設計学基礎（トップダウン：要求→機能→要素）と本講義（ボトムアップ：要素→構造→全体）は**補完関係**。

---

## 8. 構造 vs 動的モデリング

| | 構造モデリング（SM） | 動的モデリング（DM） |
|---|------------------------|----------------------|
| 焦点 | 要素間関係・階層 | 時間変化・フィードバック |
| 代表手法 | ISM, DEMATEL, Cognitive Map | システムダイナミクス |
| 役割 | 整理・合意・本質把握 | 挙動予測・政策実験 |
| 問い | 「何が何に影響するか」 | 「時間とともにどう変わるか」 |
