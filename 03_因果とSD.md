## 2. Cognitive Map（認知地図）

| 項目 | 内容 |
|------|------|
| 提唱 | Axelrod (1976) |
| 表現 | 符号付き有向グラフ **{+, −, 0}** |
| 間接効果 | パス上の符号の**積** |
| 限界 | 複数パスで符号が矛盾 → **不確定（indeterminate）** |
| 用途 | 因果の**方向**、トレードオフの可視化、議論の初期段階 |

### 隣接行列での分析

- \(A^2\) の \((i,j)\) = 2ステップパスの符号積の和
- 単一パスなら厳密；複数パスは等重み加算の仮定が必要
- 厳密な強さ比較 → **DEMATEL**

### スマートフォン5要素例（講義）

画画面・処理性能・バッテリー・携帯性・価格 — トレードオフの媒介・犠牲者を表で整理。

## 4. 因果ループ図

### 記法

| 記号 | 意味 |
|------|------|
| **S** (Same) | 原因↑→結果↑、原因↓→結果↓ |
| **O** (Opposite) | 原因↑→結果↓ |
| **R** (Reinforcing) | 強化ループ — ループ内の **Oが偶数個** |
| **B** (Balancing) | バランスループ — ループ内の **Oが奇数個** |

### 判定例

「ストレス↑→(S)対症療法↑→(O)ストレス↓」→ Oが1個 → **Bループ**

### 行列表現

ループの符号 = 辺の S/O の積。\(A^n\) の対角成分で n 次ループを検出可能（講義の発展）。

## 5. 8つのシステム原型（試験頻出）

| # | 日本語 | 英語 | 構造の要点 |
|---|--------|------|------------|
| ① | 応急処置の失敗 | Fix That Fail | B:症状軽減 + R:意図せぬ悪化（遅延） |
| ② | 問題の転嫁 | Shifting the Burden | 対症療法が根本解決能力を蝕む |
| ③ | 成功の限界 | Limits to Success | R:成長 + B:制約の顕在化 |
| ④ | 漂流する目標 | Drifting Goals | 目標を下げてギャップ解消 |
| ⑤ | 成功には成功を | Success to the Successful | 資源が成功者に偏る |
| ⑥ | エスカレート | Escalation | 相互の脅威認識が増幅 |
| ⑦ | 共有地の悲劇 | Tragedy of the Commons | 共有資源の過剰利用 |
| ⑧ | 成長と投資不足 | Growth and Underinvestment | 能力投資不足が成長を阻害 |

各原型は **Rループ（強化）とBループ（平衡）の組み合わせ**として理解する。

## 6. システムダイナミクス（SD）

### 3要素

| 要素 | 記号 | 例 |
|------|------|-----|
| **ストック** | 矩形 | 人口、資本、在庫 |
| **フロー** | 矢印（バルブ） | 出生率、投資 |
| **コンバータ** | 円 | 補助変数・関数 |

### 基礎モデル

- 一次関数、指数増加、指数減衰、収束（ネガティブフィードバック）

### 因果ループ → SD 変換

- 変数が**蓄積**する → ストック
- **率**で変化する → フロー
- 「応急処置」がフローかコンバータかは文脈で判断（講義：友人雇用数→フロー例）
