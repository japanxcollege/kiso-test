## １．講義で理解できた内容，興味を持った内容

第2回の補足として、可到達行列の収束判定（Mⁿ = Mⁿ⁻¹ で停止）、ブロック三角化（強連結成分のまとめ）、骨格行列（推移律で導出できる冗長辺の除去）、ISM/DEMATEL の使い分け（強弱の区別・目的が階層か影響度か）が整理できた。第1回のグラフ・行列が、今回の因果分析の土台になっている流れが明確だった。

Cognitive Map（認知地図）は、Axelrod (1976) に基づき概念をノード、因果的主張を符号付き有向辺 (+)/(−) で表す手法だと理解した。単一パスの間接効果はエッジ符号の積で厳密に決まり、複数パスで符号が一致すれば総合効果が確定する。矛盾する場合は「不確定」であり、これが重み付き CM → FCM → DEMATEL へ発展する理由だとつながった。中東の食糧自給率の例では、文章から変数抽出→因果特定→グラフ→隣接行列という一連の手順が具体的だった。

スマートフォン5要素のトレードオフ分析では、直接関係の符号付き隣接行列に加え、A² で間接パス（画面→バッテリー→携帯性 = (−)×(−) = (+) など）が見つかり、直接の (−) と矛盾しうる点が印象的だった。行和 D・列和 R で「トレードオフを与える側／受ける側」を読む方法は、DEMATEL の D−R, D+R と対応づけて理解できた。

因果ループ図の S/O 記法と R/B ループ（ループ内の O の個数が偶数→R、奇数→B）は、ストレスと対症療法の例で手計算できた。8つのシステム原型（応急処置の失敗、問題の転嫁、成功の限界など）は、R と B の組み合わせパターンとして名前と構造を対応づけられるようになった。

システムダイナミクス（SD）では、ストック（蓄積）・フロー（率）・コンバータの3要素と、因果ループ図が「コンセプトスケッチ」、SD が「量とメカニズム」を与える道具だという位置づけが分かった。人口モデル（出生＝人口×出生率、死亡＝人口×死亡率）と SIR モデルは、因果ループの R/B を式に落とす典型例として有用だった。因果ループ図の符号付き行列分析は循環構造では級数が発散しうるため、定量予測は SD 変換が正道、という限界の整理も納得できた。

## ２．理解できなかった内容，不明な点

Cognitive Map で複数パスが矛盾するとき、A² の算術和の正負で判断する実務と、Axelrod の「不確定」の定義の使い分けが、試験ではどちらを優先すべきかまだ曖昧である。

因果ループ図を SD に変換するとき、「遅延」の時間スケール（何日・何年）を最初からどう置くか、講義例では定性的に触れられたが、パラメータ推定の手順が不足している。

符号付き隣接行列を因果ループ図に適用する際、DEMATEL の正規化 D(I−D)⁻¹ を符号付き行列にそのまま当てはめられない理由（固有値・スペクトル半径）は概論は分かったが、方法 A/B/C のうち実務でどれを選ぶかの判断基準が不明である。

8原型の識別は名前は覚えられるが、複合モデル（製品競争など）で複数ループが絡むとき、どのループを優先して政策介入点にするかの読み方がまだ弱い。

Insight Maker で SIR モデルを組む際、感染フローを「ストック同士の矢印」から「中間フロー」に分解する具体式（接触率の置き方）を、講義外の文献なしで自力で書ける自信がない。

### 小演習：システムダイナミクス（Insight Maker）

**課題（スライド記載）**  
1. 人口モデルをモデリング  
2. シミュレーション実行  
3. パラメータ（出生率・死亡率）を変更して挙動観察  
4. 基礎モデル（一次関数、指数増加、指数減衰、収束）も試す  

**モデル構成（人口モデル）**

| 要素種別 | 変数名 | 関係・式 |
|----------|--------|----------|
| ストック | 人口 | 初期値 1000（例） |
| フロー（流入） | 出生数 | 人口 × 出生率 |
| フロー（流出） | 死亡数 | 人口 × 死亡率 |
| コンバータ | 出生率 | 0.02（年⁻¹ など時間単位に合わせる） |
| コンバータ | 死亡率 | 0.01 |

**因果ループとの対応**

- 出生数 →(S) 人口 →(S) 出生数：**R（強化ループ）**
- 死亡数 →(O) 人口 →(S) 死亡数：**B（バランスループ）**

**シミュレーション結果の整理**

| 条件 | 挙動 | 講義との対応 |
|------|------|----------------|
| 出生率 > 死亡率（例 0.02 > 0.01） | 人口が指数関数的に増加 | R が支配 |
| 出生率 < 死亡率（例 0.01 < 0.02） | 人口が指数関数的に減衰 | B が支配 |
| 出生率 = 死亡率 | 人口は初期値付近で均衡 | 流入＝流出 |

出生率を上げると増加の傾きが急になり、死亡率を上げると減少または均衡への収束が早まる、というパラメータ感度を確認できた。

**基礎モデル（講義スライドの4類型）**

| モデル | FB | 挙動の確認 |
|--------|-----|------------|
| 一次関数 | なし | ストックが時間に対して直線的に増減 |
| 指数増加 | R | フロー ∝ ストック → 加速度的増加 |
| 指数減衰 | B | フロー ∝ ストック（負のフィードバック）→ 減少 |
| 収束 | B | フロー ∝ (目標 − ストック) → 目標値へ接近 |

### 例題1：4ノードの Cognitive Map と行列計算

**構造**  
A(政策) →(+) B(経済成長) →(+) C(雇用) →(−) D(犯罪率)  
A →(−) D、B →(−) D（直接関係あり）

**符号付き隣接行列 A**（行→列：A, B, C, D）

|   | A | B | C | D |
|---|---|---|---|---|
| A | 0 | +1 | 0 | −1 |
| B | 0 | 0 | +1 | −1 |
| C | 0 | 0 | 0 | −1 |
| D | 0 | 0 | 0 | 0 |

**A² の主な読み取り（講義スライドと同様）**

- A²(A,C) = (+1)(+1) = **+1** → 政策は雇用を間接的に増加させる  
- A²(A,D) 経由 B：(+1)(−1) = **−1**；直接 A→D も **−1**  
- A→D へのパス：A→B→D (−)、A→B→C→D (−)、A→D (−) → **全パス (−) で一致** → 総合効果は犯罪率**低下**（−）と確定

### 例題2：スマートフォン5要素のトレードオフ（Step 1〜2）

**要素**  
①バッテリー容量 ②画面サイズ ③処理性能 ④携帯性 ⑤価格の安さ  

**問い**  
要素間の因果（トレードオフ）を (+)/(−)/(0) で表す。

**符号付き隣接行列（行 i ↑ → 列 j の変化）**

| 行＼列 | ①電池 | ②画面 | ③性能 | ④携帯 | ⑤価格 |
|--------|-------|-------|-------|-------|-------|
| ①電池 | 0 | 0 | 0 | −1 | −1 |
| ②画面 | −1 | 0 | 0 | −1 | −1 |
| ③性能 | −1 | 0 | 0 | 0 | −1 |
| ④携帯 | 0 | 0 | 0 | 0 | +1 |
| ⑤価格 | 0 | 0 | 0 | 0 | 0 |

**読み取り（講義の Step 2）**

- 行の (−) が多い要素：**画面サイズ（3個）**がトレードオフを最も引き起こす  
- 列の (−) が多い要素：**価格の安さ**が最も制約を受ける（犠牲者）  
- 携帯性 → 価格 (+)：軽量化は価格に好影響の例  

**Step 3（間接）の例**  
A²(画面, 携帯) = (−1)×(−1) = **+1**（画面↑→電池↓→携帯↑の間接経路）だが、直接は画面→携帯 (−) もある → **総合は不確定または定量（DEMATEL）が必要**（講義の注意事項どおり）。

### 例題3：因果ループ — ストレスと対症療法

**リンク**  
仕事のストレス →(S) 対症療法の利用 →(O) 仕事のストレス（遅延）

**ループ判定**  
ループ内の O は **1個（奇数）** → **バランスループ B**  

**解釈**  
対症療法でストレスは一時的に下がるが、根本解決にならず「問題の転嫁」原型に近い。行列表現では ストレス→対症 S、対症→ストレス O → ループ符号 S×O = **(−)** → B。

### 例題4：因果ループ → SD 変換（応急処置の失敗・起業家例）

**因果ループ（抽象）**  
問題の症状 →(S) 応急処置 →(O) 症状（B）  
応急処置 →(S) 意図せざる結果 →(S) 症状（R）

**SD への具体化（講義スライドの例）**

| 抽象変数 | SD 上の解釈 |
|----------|-------------|
| 問題の症状 | **ストック**：人手不足の程度 |
| 応急処置 | **フロー**：友人の雇用数（従業員ストックへの流入） |
| 意図せざる結果 | **ストック**：製品品質の低下 |

「何が・どれだけ・溜まるか」を問い直すことが変換の本質、という講義の指針に沿った対応。

### 例題5：人口モデルの SD 方程式（講義記載）

```
d(人口)/dt = 出生数 − 死亡数
出生数 = 人口 × 出生率
死亡数 = 人口 × 死亡率
```

出生率 > 死亡率 → 指数増加；< → 指数減衰；= → 均衡。
