## A. 構造分析（青山担当）

### 科目全体の背景

現実の社会問題（環境、人口、まちづくり、組織課題）は **多くの要素が複雑に絡む**：要素同士の関係が階層的だったり、因果が循環したり、影響度が異なったり。これを **構造化** して可視化・分析するための手法群を扱う。

主な手法：
- **ISM**：階層化（推移的な依存関係）
- **DEMATEL**：影響度の定量化（原因/結果の分離）
- **Cognitive Map / FCM**：符号付きの因果
- **AHP**：複数基準下の優先順位
- **SD（システムダイナミクス）**：時間発展シミュレーション

これらは「**悪定義問題（wicked problem）**」（目的・関係者・解が曖昧で合意形成が困難な問題）を整理するためのツール。

### A-1：システムの定義

#### 背景

「システム」という言葉は日常多用されるが、工学では厳密な定義が必要。**Bertalanffy（1968）の一般システム理論**が出発点で、「**要素 + 相互作用 + 目的**」の3要素として整理される。

また、**境界の引き方**（存在論 vs 認識論）は、システム分析の出発点として重要。同じ対象でも、分析者の問題意識によって境界が変わる。

#### 必要な知識

1. **システムの3要素**：要素、相互作用、目的性
2. **境界の二側面**：存在論（客観的実在）vs 認識論（分析者が設定）
3. **SM（Structural Modeling）vs DM（Dynamic Modeling）**：構造の静的把握 vs 時間発展の動的把握
4. **良定義 vs 悪定義（wicked problem）**：Rittel & Webber (1973)

#### 問1 システムの3要素

**模範解答**：

- **要素（components）**：システムを構成する実体・部分（人、機械、概念、組織単位など）
- **相互作用（interaction）**：要素間の関係・因果・物質・情報・エネルギーの流れ
- **目的性（purpose）**：システム全体が達成しようとする機能・目標（生物では生存・繁殖、組織では業績・存続）

#### 問2 存在論的 vs 認識論的境界

| 観点 | 内容 | 例 |
|------|------|----|
| **存在論的** | 客観世界に**実在する**区切り | 細胞膜、国境線、企業の法人格 |
| **認識論的** | 分析者が**モデル化のために設定**した区切り | 「学生」を分析するときに「経済的に独立した者を除く」など |

**核心**：同じ対象でも、分析の観点・目的によって認識論的境界は変わる。だから「**境界の設定は分析の出発点**」であり、これを明示することが重要。

#### 問3 SM vs DM

| | SM（Structural Modeling） | DM（Dynamic Modeling） |
|---|-----|-----|
| 焦点 | **構造・関係** | **時間変化** |
| 代表的手法 | ISM, DEMATEL, Cognitive Map, AHP | SD（System Dynamics）, ABM |
| 目的 | 整理・合意形成・階層把握 | 挙動予測・政策実験・シナリオ分析 |
| 時間軸 | 静的 | 動的 |

#### 問4 良定義 vs 悪定義（wicked problem）

- **良定義問題**：目標・手段・評価基準が明確（例：最短経路問題、線形計画）。最適解が存在・計算可能。
- **悪定義問題（wicked problem）**：目的・関係者・解が曖昧で合意形成が困難。例：気候変動、貧困、まちづくり。**解は「正しい/間違い」ではなく「より良い/悪い」で評価**。構造化手法が有効。

#### 採点のポイント

- 問1：**3要素を明確に分けて**書く
- 問2：「**観点で変わる**」と書ければ加点
- 問3：表形式が分かりやすい
- 問4：「**合意形成**」「**評価基準の曖昧さ**」を含む

#### 関連事項

- **Checkland の Soft Systems Methodology（SSM）**：悪定義問題への代表的アプローチ
- **CATWOE 分析**：Customer, Actor, Transformation, Worldview, Owner, Environmental constraints
- **Open vs Closed system**：環境とのやりとりの有無

### A-2：グラフと可到達行列

#### 背景

要素間の関係を**有向グラフ**で表現し、その**隣接行列 A**で計算する。さらに **可到達行列 R** を計算することで、「i から j に到達可能か」が直接読み取れる。これが ISM 法の数学的基盤。

#### 必要な知識

1. **隣接行列 A**：a_{ij} = 1 ⇔ i → j のエッジあり
2. **行列の冪と経路**：A^k の (i,j) 成分 = i から j への長さ k の経路数
3. **可到達行列 R**：i から j に**何かしらの長さで**到達可能か
4. **M = A + I の冪乗**：自分自身も到達可能とみなす（自己ループ）
5. **収束**：n ノードでは n−1 回程度で M^k = M^{k+1} になる

#### 問1 a_{ij} の意味

**模範解答**：a_{ij} = 1 はノード i から j へ**有向エッジ（影響・到達関係）が存在**することを意味する。**i が原因、j が結果**の方向。

#### 問2 可到達行列を求めるための M = A + I の冪乗

**M = A + I** は「自己ループを加えた隣接行列」。

- M の (i,j) 成分 = i から j への長さ 0〜1 の経路（0 は自分自身、1 は1本のエッジ）
- M² の (i,j) 成分 = i から j への長さ 0〜2 の経路
- M^k の (i,j) 成分 = i から j への長さ 0〜k の経路

**n ノードの場合、最長の単純経路は n−1 ステップ**なので、**M^{n−1}** で可到達行列が得られる（推移律が成り立つ場合）。

実務的には M, M², M³... と計算し、**M^k = M^{k+1}** になった時点で収束したと判定する。

#### 問3 ブロック三角化・骨格行列

- **ブロック三角化**：可到達行列を、**強連結成分**（互いに到達可能なノード群）ごとにまとめ、依存関係が階層的になるよう並べ替える。これにより**上位ブロック → 下位ブロック**の構造が見える。
- **骨格行列（skeleton matrix）**：冗長な推移関係（A→B かつ B→C のとき A→C のような派生関係）を除いた**最小の構造関係**。ISM 図の可読性を上げる。

#### 採点のポイント

- 問2：**n−1 回**で収束、計算手順
- 問3：**強連結成分** や **冗長関係の除去**といったキーワード

### A-3：ISM 法

#### 背景

**ISM（Interpretive Structural Modeling, Warfield 1973）**：要素間の **推移的な依存関係** を仮定し、要素を**階層図**に整理する手法。組織のビジョン展開、政策目標体系の構造化に使われる。

「**この要素は他のどの要素より上位（根本原因）か**」を一対比較で集めて行列にし、推移閉包を取って階層を抽出する。

#### 必要な知識

1. **推移律**：s_i R s_j ∧ s_j R s_k ⇒ s_i R s_k
2. **到達可能集合 R(i)**：i から到達可能な要素の集合
3. **先行集合 A(i)**：i に到達可能な要素の集合（i の上位側）
4. **レベル抽出条件**：**R(i) ∩ A(i) = R(i)** の要素は最上位レベル
5. **ISM vs DEMATEL**：関係の表現と推移律の扱いが異なる

#### 問1 推移律の定義と反例

**定義**：関係 R が推移的とは、**s_i R s_j かつ s_j R s_k ならば s_i R s_k** が常に成り立つこと。

**ISM が推移律を要求する**理由：階層化のためには「上位は上位、下位は下位」の連鎖が論理的に整合している必要があるため。

**反例**：
- **「友人関係」**：A は B の友人、B は C の友人。しかし A と C が友人とは限らない
- **「じゃんけん」**：グー > チョキ、チョキ > パー だが、グー < パー
- **「相性」**：A は B と相性が良い、B は C と相性が良い、しかし A と C は悪い

#### 問2 レベル抽出条件 R ∩ A = R の意味

各レベル（階層）を抽出するアルゴリズム：

- **R(i)**：i から到達可能な要素の集合（i 以下、下流）
- **A(i)**：i に到達可能な要素の集合（i 以上、上流）
- **R(i) ∩ A(i)**：i と同レベルで互いに到達可能な要素

**条件 R(i) ∩ A(i) = R(i)** は、

> 「i から到達できる要素は、すべて i にも到達できる」

を意味する。これは「**i 以下に i より下流の要素はもう無い**」＝**i は最上位ブロック**を意味する。

**アルゴリズム**：
1. 全要素について R(i) ∩ A(i) = R(i) を満たす要素を**最上位レベル**として抽出
2. 抽出した要素を除いた残りの要素で同じ判定を繰り返す
3. すべての要素が階層化されるまで反復

#### 問3 ISM vs DEMATEL

| | ISM | DEMATEL |
|---|-----|---------|
| 関係の表現 | **0/1**（あるかないか） | **0〜4**（影響の強さ） |
| 推移律 | **仮定する** | 仮定しない |
| 入力データ | 対話的一対比較 | アンケート、専門家評価 |
| 主目的 | **階層化** | **影響度・原因/結果の定量** |
| 出力 | 階層図 | D−R 図、D+R 図 |
| 適用例 | 政策目標体系、KPI 構造化 | 経営課題、リスク要因分析 |

#### 採点のポイント

- 問1：**推移律の式**と**具体的な反例**
- 問2：R と A の定義、**条件の意味（最上位ブロック）**
- 問3：**4点の比較表**（関係、推移律、入力、目的）

#### 関連事項

- **TISM（Total ISM）**：影響の強さも考慮した拡張
- **MICMAC 分析**：駆動力・依存度で要素を4象限に分類
- **DEMATEL との併用**：DEMATEL で重要要素を抽出 → ISM で階層化

### A-4：DEMATEL 法（計算）

#### 背景

**DEMATEL（Decision Making Trial and Evaluation Laboratory）**：Battelle Geneva 研究所が 1972 年に開発。複雑な要因系の **直接 + 間接影響** を行列計算で集計し、**原因系（D−R > 0）と結果系（D−R < 0）**に分離する。

ISM が「ある/なし」のバイナリ階層化なのに対し、DEMATEL は**影響の強さ**を 0〜4 で評価し、**間接効果も含めた総合影響**を計算する。

#### 必要な知識

1. **直接影響行列 X**：x_{ij} = i が j に与える直接影響度（0〜4）
2. **正規化**：N = X / s, ただし s = 各行の行和の最大値（または列和の最大値）
3. **総合影響行列 T = N(I−N)^{-1} = N + N² + N³ + ...**：直接 + 間接の総和
4. **D（行和）**：i が他要素に与える総影響量（**発信力**）
5. **R（列和）**：i が他要素から受ける総影響量（**被影響力**）
6. **D + R**：要素 i の**関連度**（システム内の重要性）
7. **D − R**：要素 i の**因果性**（>0 なら原因系、<0 なら結果系）

#### 問題設定

3要素の直接影響行列：

$$
X = \begin{pmatrix}
0 & 2 & 0 \\
0 & 0 & 3 \\
0 & 0 & 0
\end{pmatrix}
$$

#### 問1 正規化行列 N

各行の行和：
- 1行目：0+2+0 = **2**
- 2行目：0+0+3 = **3**
- 3行目：0+0+0 = **0**

最大行和 = **3** で割る：

$$
N = \frac{X}{3} = \begin{pmatrix}
0 & 2/3 & 0 \\
0 & 0 & 1 \\
0 & 0 & 0
\end{pmatrix}
$$

#### 問2 総合影響行列 T

**ポイント**：N は **冪零**（N³ = 0）なので、(I−N)^{-1} = I + N + N² と有限和で表せる。

N²を計算：

$$
N^2 = \begin{pmatrix}
0 & 2/3 & 0 \\
0 & 0 & 1 \\
0 & 0 & 0
\end{pmatrix}^2 = \begin{pmatrix}
0 & 0 & 2/3 \\
0 & 0 & 0 \\
0 & 0 & 0
\end{pmatrix}
$$

N³ = 0。よって：

$$
(I - N)^{-1} = I + N + N^2 = \begin{pmatrix}
1 & 2/3 & 2/3 \\
0 & 1 & 1 \\
0 & 0 & 1
\end{pmatrix}
$$

T = N(I−N)^{-1} を計算：

$$
T = N + N^2 + N^3 = N + N^2 = \begin{pmatrix}
0 & 2/3 & 2/3 \\
0 & 0 & 1 \\
0 & 0 & 0
\end{pmatrix}
$$

**直接影響だけだと「①→③」は 0** だが、総合では **①→②→③** という**間接影響 2/3** が現れる。これが DEMATEL の意義。

#### 問3 D, R, D±R の計算と原因/結果系の判定

**D（行和）**：
- ①：0 + 2/3 + 2/3 = **4/3**
- ②：0 + 0 + 1 = **1**
- ③：0 + 0 + 0 = **0**

**R（列和）**：
- ①列：0 + 0 + 0 = **0**
- ②列：2/3 + 0 + 0 = **2/3**
- ③列：2/3 + 1 + 0 = **5/3**

| 要素 | D | R | D+R | D−R | 判定 |
|------|---|---|-----|-----|------|
| ① | 4/3 | 0 | **4/3** | **+4/3** | 最も**原因系**（発信のみ） |
| ② | 1 | 2/3 | **5/3** | **+1/3** | やや原因系（中継的） |
| ③ | 0 | 5/3 | **5/3** | **−5/3** | 最も**結果系**（受信のみ） |

**解釈**：
- ① は他要素に影響を与えるだけで受けない → **介入すべき根本原因**
- ③ は影響を受けるだけ → **対症療法すべき症状**
- ② は中間的 → ①→②→③ の連鎖の中継点

#### 採点のポイント

- 問1：**最大行和で除算**を明示
- 問2：**(I−N)^{-1} の計算**（N が冪零なら有限和）、T の最終形
- 問3：**D, R の計算過程**、D+R と D−R を別表で書く、原因/結果系の解釈

#### 関連事項

- **NRM（Network Relation Map）**：D−R を縦軸、D+R を横軸にプロット
- **FUZZY DEMATEL**：評価の曖昧さをファジィ数で扱う拡張
- **応用例**：サプライチェーンリスク、エネルギー政策、教育評価

### A-5：因果ループ・SD（システムダイナミクス）

#### 背景

**SD（Forrester 1961）**：MIT の Jay Forrester が産業動態のために開発。**フィードバックループ**を中核に置き、**ストック・フロー**で時間発展をモデル化する。気候変動の **World3 モデル**、組織学習の **Senge の第5の規律** の数学的基盤。

**因果ループ図（CLD: Causal Loop Diagram）** は SD の質的記法で、要素間の**正負の影響**と**ループの種類**（強化 R / バランス B）を視覚化する。

#### 必要な知識

1. **S/O 記号**：S（Same）= 同方向変化、O（Opposite）= 逆方向変化
2. **強化ループ（R: Reinforcing）**：自己強化・発散傾向。**O の個数が偶数**
3. **バランスループ（B: Balancing）**：均衡・収束傾向。**O の個数が奇数**
4. **ストック（stock）**：時点での蓄積量（状態変数）。例：人口、貯金、在庫
5. **フロー（flow）**：単位時間あたりの増減（変化率）。例：出生、入金、生産
6. **遅延（delay）**：因果が時間差で効く
7. **システム原型（systems archetype）**：Senge が整理した典型的ループパターン

#### 問1 S と O の意味

- **S（Same）**：原因が増えれば結果も増える（同方向）。例：勉強時間 ↑ → 学力 ↑
- **O（Opposite）**：原因が増えれば結果は減る（逆方向）。例：価格 ↑ → 需要 ↓

#### 問2 ループ判定

連鎖：「**ストレス↑ →(O) 睡眠↓ →(O) 疲労↑ →(S) ストレス↑**」

- O の個数：**2 個（偶数）**
- 判定：**強化ループ R**

**実際にトレース**：ストレス↑ → 睡眠↓ → 疲労↑ → ストレス↑（さらに増加）→ ...

→ **自己強化の悪循環**。ストレス・不眠・疲労の典型的なメンタルヘルス悪化パターン。

**一般則**：
- O が**偶数**個 → 全体として同方向（自己強化）→ **R**
- O が**奇数**個 → 全体として逆方向（均衡作用）→ **B**

数学的には：S = +1, O = −1 と置き、ループ全体の積が **+1 → R**、**−1 → B**。

#### 問3 ストック vs フロー

- **ストック**：時点で測れる**蓄積量**（積分量）。状態変数。例：人口、湖の水量、CO₂濃度、口座残高。
- **フロー**：単位時間あたりの**増減量**（微分量）。例：出生率、流入量、排出率、入金額。

**関係**：ストックの時間微分 = 流入フロー − 流出フロー（連続方程式）。

ストックは**過去の蓄積**を持つので、システム挙動の**慣性・遅延**を生む。SD でストックを正しく特定することが**システム理解の核心**。

#### 問4 Fix That Fail（応急処置の失敗）

**システム原型のひとつ**。Senge の整理による。

| ループ | 役割 |
|--------|------|
| **B ループ（即効）** | 症状 → 応急処置 → 症状の一時的軽減（バランス） |
| **R ループ（副作用）** | 応急処置 → 遅延 → 別の問題悪化 → 元の症状悪化（強化） |

**例**：
- **借金で生活費を補填**：当面は楽（B）→ 利息で長期的に困窮（R）
- **抗生物質乱用**：感染症が治る（B）→ 耐性菌が増える（R）
- **道路拡張で渋滞緩和**：一時的に流れる（B）→ 誘発交通で混雑悪化（R）

**示唆**：応急処置は**短期的解決と長期的悪化のトレードオフ**を生む。**根本原因への介入**が必要。

#### 採点のポイント

- 問1：**S = 同方向、O = 逆方向**を1文で
- 問2：**O の個数で判定**するルール、**R/B の名前**
- 問3：**蓄積 vs 変化率**の対比、**慣性・遅延**を生む
- 問4：**B（症状軽減）と R（根本悪化）の両方**、具体例

#### 関連事項

- **World3 モデル**（Limits to Growth, 1972）：人口・資源・汚染・食糧・工業の世界モデル
- **Senge の第5の規律**：組織学習における5つの規律のひとつ「システム思考」
- **他のシステム原型**：Limits to Growth, Shifting the Burden, Tragedy of the Commons, Eroding Goals
- **Stella, Vensim**：SD シミュレーションソフト

### A-6：ファジィ理論・AHP

#### 背景

**ファジィ理論（Zadeh 1965）**：「**曖昧さ**」を数学的に扱う。確率（発生の不確実性）とは異なる、**概念の境界の曖昧さ**を扱う。「高齢」「速い」「美味しい」など。

**AHP（Analytic Hierarchy Process, Saaty 1980）**：多基準意思決定の代表手法。階層構造（目標 → 基準 → 代替案）を作り、**一対比較**で重みを求め、総合評価する。

#### 必要な知識

1. **メンバシップ関数 μ_A(x) ∈ [0,1]**：x が集合 A に属する度合い
2. **確率 vs ファジィ**：発生の不確実性 vs 概念の曖昧さ
3. **AHP の階層構造**：目標 → 基準 → 代替案
4. **一対比較行列**：Saaty の 1〜9 スケール
5. **整合性指標 CR（Consistency Ratio）**：判断の一貫性を検証、**CR < 0.1** が目安

#### 問1 ファジィ vs 確率

| | 確率 | ファジィ |
|---|------|----------|
| 対象 | **事象の発生**の不確実性 | **概念の境界**の曖昧さ |
| 例 | 雨が降る確率 60% | 「高齢」の度合い 0.7 |
| 数学的基盤 | Kolmogorov 公理 | 集合論の拡張 |
| 解釈 | 何度も繰り返したら 60% で起きる | 「高齢」という概念に 70% 属する |

**両者は別の不確実性を扱う**。確率では「雨が降る/降らない」は二択（起きるか起きないかが確定すれば白黒）。ファジィでは「高齢」は本質的にグラデーション。

#### 問2 AHP の一対比較と CR

- **一対比較**：基準同士、または代替案同士を**2つずつ取り出して相対的重要度を 1〜9 で評価**する。例：「環境 vs コスト」で環境が「やや重要 = 3」。
- **比較行列**：n×n の正逆対称行列（a_{ji} = 1/a_{ij}）から固有ベクトルで重みを求める
- **整合性指標 CR**：判断が矛盾していないかを検証。
  - 例：A > B、B > C なのに C > A としていれば矛盾
  - CR = CI / RI（CI: 整合性指標、RI: ランダム指標）
  - **CR < 0.1** で「一貫している」とみなす。超えたら判断を見直す

#### 採点のポイント

- 問1：**対象（発生 vs 概念）**を区別
- 問2：**一対比較**と**CR の役割**を1文ずつ、**0.1 の目安**

#### 関連事項

- **Fuzzy AHP**：両者を組み合わせ
- **ANP（Analytic Network Process）**：階層に限らないネットワーク構造
- **TOPSIS**：別の多基準意思決定法（理想解からの距離）

### A-7：手法の使い分け

#### 背景

ここまで学んだ複数の手法を、状況に応じて選び分ける応用力を問う問題。**手法は道具であり、問題に応じて選ぶ**ことが工学的思考。

| 状況 | 望まれる出力 | 適した手法 |
|------|--------------|------------|
| 階層あり、推移律あり | 階層図 | **ISM** |
| 階層なし、影響度・原因結果 | D−R 図 | **DEMATEL** |
| 因果の符号（増減）が重要 | 因果ループ図 | **Cognitive Map** |
| 複数代替案の優先順位 | 重み・順位 | **AHP** |
| 時間発展の挙動 | 時系列 | **SD** |

#### 模範解答

| | 手法 | 理由 |
|---|------|------|
| (a) SNSフォロー関係、影響の強さが欲しい | **DEMATEL** | フォロー関係は推移律が成り立たない（A→B, B→C でも A→C とは限らない）。影響度の定量が目的 |
| (b) 製品評価基準の下で3案の優先順位 | **AHP** | 複数基準・複数代替案の総合評価が目的。一対比較で重みを決められる |
| (c) 要因A が要因B を「増やす/減らす」の符号が本質 | **Cognitive Map** | 正負の因果が分析対象。ISM や DEMATEL は符号を扱わない |
| (d) 影響が推移的、上位下位の階層が欲しい | **ISM** | 推移律と階層化が定義通り |
| (e) 政策介入後10年間の人口変化を試算 | **SD** | 時間発展のシミュレーションが目的。ストック（人口）とフロー（出生・死亡・転入・転出） |

#### 採点のポイント

- **各選択の理由を1文**：単に手法名だけでなく「**何が条件に合うか**」を述べる

## B. モデリング・シミュレーション（藤井担当）

### 科目全体の背景

不確実性のある現象（顧客の到着、機器の故障、市場価格、感染症の拡大）を**確率モデル**で記述し、**シミュレーション**で挙動を予測する。

主な手法：
- **マルコフ連鎖**：状態遷移を確率で記述（マルコフ性 = 過去履歴を忘れる）
- **待ち行列**：到着・サービス過程のモデル化（M/M/1, M/M/c）
- **セルオートマトン**：局所ルールから大域挙動を生成
- **自己組織化臨界（SOC）**：べき乗則とサンドパイル
- **エージェントベース・モデル（ABM）**：自律個体の相互作用

### B-1：確率・マルコフ

#### 背景

**確率過程**：状態が確率変数として時間とともに変化する系。**マルコフ性**は「**未来は現在のみに依存し、過去履歴を忘れる**」という性質で、解析が劇的に簡単になる。

実例：天気の遷移、機械の故障/修理、株価モデル、ランダムウォーク、PageRank。

#### 必要な知識

1. **確率モデル**：確率則で記述された数学的モデル
2. **確率過程**：{X_t} の集合、各 X_t は確率変数
3. **マルコフ性**：P(X_{n+1} = j | X_n = i, X_{n-1}, ..., X_0) = P(X_{n+1} = j | X_n = i)
4. **推移行列 P**：p_{ij} = P(X_{n+1} = j | X_n = i)、各行の和 = 1
5. **定常分布 π**：π P = π かつ Σπ_i = 1
6. **分布関数 F(x) = P(X ≤ x)** の3性質：単調非減少、右連続、F(−∞)=0, F(∞)=1

#### 問1 確率モデル vs 確率過程

- **確率モデル**：確率則を使って現象を記述するモデル全般（例：サイコロ、ベルヌーイ試行、正規分布モデル）
- **確率過程**：状態が**時間とともに変化**する確率モデル。{X_t}_{t∈T} の確率変数族

**関係**：確率過程は確率モデルの一種（時間軸を持つ）。

#### 問2 マルコフ性

**式**：

$$
P(X_{n+1} = j \mid X_n = i, X_{n-1} = i_{n-1}, \ldots, X_0 = i_0) = P(X_{n+1} = j \mid X_n = i)
$$

**言葉**：「**次の状態は現在の状態だけで決まり、過去の履歴は不要**」。条件付き独立性の一種。

**意義**：マルコフ性のおかげで、過去の全履歴を覚えなくても遷移確率だけで挙動を追える。**計算上扱いやすい**。多くの現象がマルコフ近似で十分。

#### 問3 定常分布の計算

推移行列：

$$
P = \begin{pmatrix} 0.9 & 0.1 \\ 0.2 & 0.8 \end{pmatrix}
$$

定常分布 π = (π₁, π₂) は π P = π を満たす。

$$
0.9 \pi_1 + 0.2 \pi_2 = \pi_1
$$

$$
0.1 \pi_1 + 0.8 \pi_2 = \pi_2
$$

第1式から：

$$
0.2 \pi_2 = 0.1 \pi_1 \quad\Rightarrow\quad \pi_1 = 2 \pi_2
$$

正規化条件 π₁ + π₂ = 1：

$$
2\pi_2 + \pi_2 = 1 \quad\Rightarrow\quad \pi_2 = \frac{1}{3}, \quad \pi_1 = \frac{2}{3}
$$

$$
\boxed{\boldsymbol{\pi} = \left(\frac{2}{3}, \frac{1}{3}\right)}
$$

**解釈**：長時間後、状態1にいる確率が 2/3、状態2にいる確率が 1/3 に収束する。**初期状態に依存しない**（既約・非周期的マルコフ連鎖の場合）。

#### 問4 分布関数の3性質

$$
F(x) = P(X \le x)
$$

1. **単調非減少**：x_1 ≤ x_2 ならば F(x_1) ≤ F(x_2)（確率は累積されるだけ）
2. **右連続**：F(x+) = F(x)（離散分布でも step の右端で連続）
3. **F(−∞) = 0、F(∞) = 1**：すべての確率を尽くす

#### 採点のポイント

- 問2：**式の右辺と左辺**を明示、「**過去の履歴は不要**」のキーワード
- 問3：**π P = π** の連立、**正規化条件**、**最終的に (2/3, 1/3)**
- 問4：**3つすべて**を1行ずつ

#### 関連事項

- **既約性・非周期性**：すべての状態に到達可能 + 周期 1。これらが定常分布の一意性を保証
- **ペロン・フロベニウスの定理**：非負行列の最大固有値の理論
- **隠れマルコフモデル（HMM）**：状態が観測できない場合
- **マルコフ決定過程（MDP）**：行動選択を含むマルコフ連鎖、強化学習の基礎

### B-2：待ち行列

#### 背景

**待ち行列理論（Erlang 1909）**：電話交換機の設計から始まった。「**顧客がランダムに到着し、サーバーが処理する**」系の数学。

応用：銀行 ATM、コールセンター、Web サーバー、製造ライン、空港搭乗ゲート、交通信号。

**ケンドール記法 A/B/c**：到着過程 / サービス過程 / サーバー数
- **M/M/1**：到着がポアソン（指数分布の到着間隔）、サービスが指数分布、サーバー1台
- **M/M/c**：サーバー c 台
- **M/G/1**：サービスが一般分布

#### 必要な知識

1. **到着率 λ**：単位時間あたりの平均到着数
2. **サービス率 μ**：単位時間あたりの平均処理数（サーバー1台が忙しいとき）
3. **利用率 ρ = λ / μ**：サーバーが忙しい時間の割合
4. **安定条件 ρ < 1**：到着 < サービスでないと行列が発散
5. **リトルの法則 L = λ W**：システム内平均人数 = 到着率 × 平均滞在時間
6. **M/M/1 の主要公式**：L = ρ/(1−ρ), W = 1/(μ−λ), 平均待ち行列長 L_q = ρ²/(1−ρ)

#### 問1 利用率 ρ の定義と ρ ≥ 1 の意味

**定義**：ρ = λ / μ（到着率 / サービス率）

**ρ ≥ 1 の意味**：到着が処理を上回るため、**待ち行列が時間とともに無限に長くなる**（発散）。系が不安定。例：ATM 機 1 台で毎分2人来るのに毎分1人しか処理できなければ、毎分 1 人ずつ列が伸びる。

**実務的示唆**：稼働率は **80% 程度に抑える**のが目安（ρ → 1 で待ち時間が急増する：ρ=0.9 で平均行列長 9、ρ=0.99 で 99）。

#### 問2 リトルの法則 L = λ W

**式**：

$$
L = \lambda W
$$

- **L**：系内の平均顧客数（**ストック**）
- **λ**：到着率（**フロー**）
- **W**：平均滞在時間

**意義**：到着過程・サービス過程の**分布によらず常に成り立つ**極めて一般的な法則。ストックとフローを結ぶ普遍的関係。

**直観的説明**：1人が平均 W 時間滞在し、λ 人/時で入ってくるなら、系内の平均人数は L = λ × W。

#### 採点のポイント

- 問1：**ρ = λ/μ**、**ρ≥1 で発散**
- 問2：**L, λ, W の意味**、「**分布によらない**」が書ければ加点

#### 関連事項

- **M/M/1 の Pollaczek-Khinchine 公式**：M/G/1 の平均待ち時間
- **Erlang B/C 公式**：呼損率、待機率
- **平均場近似**：多数サーバー系の解析
- **シミュレーション**：解析解が出ない複雑系では SimPy 等で離散事象シミュレーション

### B-3：離散モデル・SOC・エージェント

#### 背景

**連続的な現象を離散ステップで表現する**手法群。コンピュータは離散計算しかできないので、シミュレーションは本質的に離散。

- **有限オートマトン**：状態 + 入力 + 遷移関数。1体の離散系
- **セルオートマトン（CA）**：格子の各セルが**局所ルール**で同時更新。Game of Life が代表
- **自己組織化臨界（SOC, Bak 1987）**：外部調整なしに臨界状態に達する。サンドパイルが典型
- **エージェントベース・モデル（ABM）**：多数の自律個体の相互作用から**創発**

#### 必要な知識

1. **有限オートマトン**：(Q, Σ, δ, q_0, F)：状態集合、入力アルファベット、遷移関数、初期状態、受理状態
2. **セルオートマトン**：(L, S, N, f)：格子、状態集合、近傍関係、更新関数
3. **Game of Life**：Conway 1970、Birth/Survival ルール
4. **サンドパイル・モデル**：Bak-Tang-Wiesenfeld 1987
5. **べき乗則 P(x) ∝ x^{−α}**：自己組織化臨界の指紋
6. **エージェント**：環境を知覚し、目標達成のために行動する自律体
7. **MAS**：多数エージェントの相互作用系
8. **創発（emergence）**：個々のルールから集団レベルで生じる予測困難な性質

#### 問1 有限オートマトン vs セルオートマトン

| | 有限オートマトン | セルオートマトン |
|---|------|--------|
| 単位 | **1 つの離散系** | **格子状の多数セル** |
| 状態 | 1つの状態 q ∈ Q | 全セルの状態（高次元配列） |
| 更新 | 入力に応じて遷移 | **局所ルール**で同時更新 |
| 入力 | 入力列 Σ* | （初期配置と境界条件） |
| 例 | 文法解析、字句解析 | Game of Life、流体、交通流 |

#### 問2 サンドパイル更新規則

砂を一粒ずつ落とし、4近傍に崩れる単純規則：

1. **ランダムに1セル選び、値を +1**
2. **値が 4 以上なら、そのセルから 4 減らし、上下左右の4近傍に +1 ずつ加える**（**崩壊**）
3. **崩壊によって他のセルも 4 以上になれば、連鎖的に崩壊を繰り返す**。すべて 3 以下に戻るまで継続
4. 安定したら次の砂粒を落とす（1 に戻る）

#### 問3 自己組織化臨界（SOC）

**SOC**：外部から微調整しなくても、**系が自然に臨界状態（相転移点）に達する**現象。サンドパイルでは、

- 小さな崩壊から**大規模な雪崩**まで、**さまざまなスケールの雪崩がべき乗分布で発生**
- 雪崩サイズ s の頻度 P(s) ∝ s^{−α}

**示唆**：
- 地震（**Gutenberg-Richter 則**）
- 山火事の規模分布
- 金融市場の暴落
- 神経活動のニューロン雪崩
- 種の絶滅イベント

これらが**べき乗則**に従うのは、SOC が普遍的なメカニズムであることを示唆。

#### 問4 エージェント・MAS・創発

- **エージェント**：環境を知覚（perception）し、自律的に行動（action）する個体。内部状態・目標・学習能力を持つことも
- **MAS（Multi-Agent System）**：多数のエージェントが相互作用する系。**集中制御なし**でも全体としての挙動が生まれる
- **創発（emergence）**：個々の局所ルールから、**集団レベルで予測困難な大域的性質**が生じる

**創発の例**：
- **市場価格**：個々の参加者の売買 → 市場全体の価格動学
- **渋滞**：個々のドライバーの追従行動 → 自然渋滞
- **群れの飛行（boids）**：3つの単純ルール（分離・整列・凝集）で鳥群の挙動
- **アリのフェロモン経路**：個々のアリの局所判断 → 最短経路の発見
- **株式市場の暴落**

#### 問5 連続体現象を離散モデル化する理由

1. **コンピュータは離散計算しかできない**：浮動小数点も有限精度。シミュレーションは本質的に離散
2. **解析的に解けない PDE を数値解する**：偏微分方程式 → 差分方程式（FDM）/ 有限要素（FEM）

**補足理由**：
- 非線形・確率的要素を入れやすい
- 個体差・空間異質性を扱いやすい
- 創発現象を観察できる

#### 採点のポイント

- 問1：**1体 vs 格子・多数**、**局所ルールでの同時更新**
- 問2：**3ステップ**を順序通り、**連鎖崩壊**
- 問3：**外部調整なし**、**べき乗則**、**サンドパイル**
- 問4：**3用語の定義**、**創発例 1 つ**
- 問5：**離散計算**と**PDE 数値解**の2点

#### 関連事項

- **Game of Life**：単純ルールから複雑挙動（万能計算可能性）
- **Wolfram の CA 分類**：Class I〜IV（均一・周期・カオス・複雑）
- **Schelling の分居モデル**：個々の弱い選好から強い分居が創発
- **Reynolds の boids**：群れシミュレーションの古典
- **NetLogo, Mesa**：ABM プログラミング環境

### B-4：エージェント構造

#### 背景

エージェントは内部構造によって分類される。**Russell & Norvig の AI 教科書**で整理された分類が広く使われる。

#### 必要な知識

5タイプ：
1. **単純反射エージェント**：現在の知覚だけで行動（条件 → 行動）
2. **モデルベース反射エージェント**：内部に世界モデルを持つ
3. **目標ベースエージェント**：目標を持ち、達成手段を選ぶ
4. **効用ベースエージェント**：効用関数の期待値を最大化
5. **学習エージェント**：経験から改善

#### 問1 単純反射 vs 効用ベース

| | 単純反射 | 効用ベース |
|---|----------|------------|
| 内部モデル | なし | 世界モデルあり |
| 目標 | なし（IF-THEN ルールのみ） | 効用関数で多目的に最適化 |
| 不確実性 | 扱わない | 期待効用で評価 |
| 行動選択 | 現在の知覚に直接反応 | **効用の期待値を最大化** |
| 例 | サーモスタット（温度 > X なら冷房） | 自動運転（安全性・速度・燃費の効用バランス） |

**効用ベースの利点**：複数目標のトレードオフを扱える、不確実性下で合理的選択ができる。

#### 採点のポイント

- **内部モデルの有無**、**効用関数による期待値最大化**

## C. レポート練習（自分の問題への適用）

### 背景

実践的な問題解決には、**手法を知っているだけでなく、自分の問題に当てはめる力**が必要。レポート練習では、自分の関心事を構造化手法でモデル化する練習をする。

### 問1 解決したい問題の明示

**ポイント**：問題は **1文で MECE に**書く。「何が、どう、どうあるべきか」が含まれていればよい。

**例**：「都市部の若者の **投票率が低下** しているのを、**選挙制度設計を変えることで改善** したい」

### 問2 ISM のための要素列挙

**ポイント**：

- 要素は **10 個程度**
- **MECE に近い**（重複なく、漏れなく）
- **影響/被影響の両方**を意識した粒度
- **抽象度が揃っている**

**例**（投票率問題）：①若者の政治関心、②候補者情報の入手しやすさ、③投票所のアクセス、④期日前投票の周知、⑤同調圧力、⑥SNS での政治発信、⑦選挙制度の理解、⑧過去の投票経験、⑨家族の投票習慣、⑩投票時の時間コスト

**一対比較の方針**：「**i は j より上位（根本的）か**」を統一質問とし、yes/no で答える。

### 問3 DEMATEL のための注意点

1. **評価基準（0〜4）をチームで事前統一**：0=影響なし、1=弱、2=中、3=強、4=非常に強。基準が人ごとに違うと結果が無意味
2. **主観バイアスへの対策**：複数専門家の評価を集める、メンタルモデルとの比較で検証、Delphi 法的に反復

### 問4 SD のためのストック・フロー候補

**例**（地方人口減少）：
- **ストック候補**：人口、住宅ストック、雇用機会の数、産業基盤の規模
- **フロー候補**：出生 − 死亡（自然動態）、転入 − 転出（社会動態）、住宅建設 − 取り壊し、雇用創出 − 雇用喪失

### 採点のポイント

- 問1：**1文で具体的**
- 問2：**10個程度**、**抽象度が揃う**
- 問3：**事前統一**と**バイアス対策**
- 問4：**ストック 2 つ + フロー 2 つ**、ストックがフローの積分になっている

## D. 総合

### 問1 青山担当と藤井担当の要約

- **青山（構造分析）**：要素間の構造・因果を **ISM/DEMATEL/Cognitive Map/AHP/SD** で整理・可視化し、悪定義問題への合意形成と介入点の発見を支援する
- **藤井（確率・離散モデル）**：**マルコフ連鎖・待ち行列・セルオートマトン・エージェント** などで不確実性下の複雑挙動をシミュレーションし、政策効果や創発現象を予測する

### 問2 誤りの指摘：「DEMATEL は因果の正負を + と − で表す」

**誤り**：DEMATEL は **非負の影響の強さ（0〜4）** を扱う。**正負（増減）の符号は表現しない**。

**正しくは**：
- **DEMATEL**：影響の**強さ**（0〜4 の非負）
- **Cognitive Map / FCM**：影響の**符号**（+/− で増減を表す）

両者は異なる情報を扱うので、目的に応じて使い分ける。

### 問3 誤りの指摘：「マルコフ連鎖では過去の全履歴が次の状態を決める」

**誤り**：マルコフ性は「**現在の状態のみ**が次の状態を決定する」（過去の履歴は不要）。

**正しくは**：

$$
P(X_{n+1} = j \mid X_n = i, X_{n-1}, \ldots, X_0) = P(X_{n+1} = j \mid X_n = i)
$$

**過去の履歴 X_{n-1}, ..., X_0 はすべて忘れて良い**（条件付き独立）。

### 採点のポイント

- 問1：**両者を1文ずつ**、対比的に
- 問2：**強さ vs 符号**、**Cognitive Map との対比**
- 問3：**マルコフ性の式**、**過去は不要**
