## 1. システムの基本概念

### システム ⭐

**意味**：複数の要素が相互に関係を持ち、全体として一つの目的を達成する集合体。

**3 要素**：① 要素 ② 相互作用 ③ 目的性

**例**：自動車システム＝エンジン・タイヤ等（要素）＋力の伝達（相互作用）＋人を運ぶ（目的）。

**試験での出方**：「システムを構成する 3 つの要素を挙げよ」「システム概念を説明せよ」

🔗 システム要素、相互関係、目的性、システム境界、システム思考

### システム概念 ⭐

**意味**：システムを「要素・相互作用・目的を持つ統合体」として捉える考え方。

**特徴**：個々の部品の総和ではなく、**全体としての創発的振る舞い**を重視。

**試験での出方**：用語論述で必出。「システム概念を説明せよ」（2015, 2016, 2022 で出題）

### システム要素

**意味**：システムを構成する個々の実体。人・物・情報・組織単位など。

**例**：自動車のエンジン、PC のキーボード、組織の部署。

🔗 相互関係、システム概念

### 相互関係 ⭐

**意味**：要素間の影響・因果・物質や情報の流れ。

**形式**：A → B（A が B に影響する）。**有向**または**無向**。

**例**：エンジンが車輪に力を伝える、上司が部下に指示を出す。

🔗 隣接行列、可到達行列、因果ループ図

### システム境界 ⭐

**意味**：システム内外を区切る境界線。「何をシステムの内部とし、何を外部とするか」の決定。

**2 種類**：
- **存在論的境界**：客観的に実在する区切り（細胞膜、国境、企業の法人格）
- **認識論的境界**：分析者がモデル化のために設定する区切り（観点で変わる）

**重要性**：境界の引き方が分析の出発点。

**試験での出方**：「存在論的と認識論的の違い」

### システムの認識論 ⭐

**意味**：システムを「どう見るか」「どこで区切るか」の議論。分析者の観点が結果を左右する。

**例**：「学生」を分析するとき、経済的に独立した者を含めるか、年齢で区切るかなど。

🔗 システム境界、メンタルモデル

### システム思考 ⭐

**意味**：要素間のフィードバックや相互依存を考慮する全体論的思考法。線形思考の対極。

**特徴**：
- 部分の足し算ではなく、創発的振る舞いを重視
- フィードバックループを意識
- 遅延・副作用を予測

**例**：渋滞対策で道路を広げても、誘発交通で再び渋滞（線形思考では予測困難）。

**試験での出方**：2023 で 15/50 点。用語論述で頻出。

🔗 因果ループ図、システムダイナミクス、メンタルモデル

### メンタルモデル

**意味**：個人が世界を捉えるための内部表象。Senge「第 5 の規律」の中核概念。

**特徴**：無意識的な前提・思い込みが行動を制約する。

**例**：「成長は無限に続く」という前提が政策判断を歪める。

🔗 システム思考、Senge

### マン・マシン系 ⭐

**意味**：人間と機械がインターフェースを介して相互作用するシステム。人間も要素として扱う。

**要素**：人間・機械・インターフェース・環境・タスク

**例**：自動車運転（ドライバー＋車）、PC 操作（ユーザー＋PC）、航空機操縦。

**試験での出方**：2022 で用語論述に登場。

### 良定義問題

**意味**：目標・手段・評価基準が明確な問題。最適化やシミュレーションで解ける。

**例**：最短経路問題、線形計画。

### 悪定義問題（wicked problem）

**意味**：目的・関係者・解が曖昧で合意形成が困難な問題。Rittel & Webber 1973。

**特徴**：
- 何が「正しい解」か関係者で意見が分かれる
- 解は「より良い／悪い」で評価
- 問題自体がやってみないと分からない

**例**：気候変動対策、貧困問題、まちづくり。

🔗 ソフトシステムズ方法論、構造化手法

### 構造モデリング（SM）⭐

**意味**：要素間の関係・階層を**静的**に整理することに焦点。時間軸を持たない。

**代表手法**：ISM、DEMATEL、Cognitive Map、AHP

**問い**：「何が何に影響するか」「どんな構造になっているか」

🔗 動的モデリング（DM）

### 動的モデリング（DM）⭐

**意味**：要素の状態が**時間とともにどう変わるか**に焦点。フィードバック・遅延を扱う。

**代表手法**：システムダイナミクス（SD）、エージェントベースモデル

**問い**：「時間とともにどう変化するか」「政策介入の長期影響は」

🔗 構造モデリング（SM）、システムダイナミクス

### SoS（System of Systems）

**意味**：独立して運用される複数のシステムが連携した上位の系。

**5 特性**：
1. 運用上の独立性
2. 管理上の独立性
3. 地理的分散
4. 創発的振る舞い
5. 進化的発展

**例**：スマートシティ、サプライチェーン、航空交通管制。

### V-Model

**意味**：要求→設計→実装 ↔ 検証・評価が対応する V 字型の設計プロセス。

**特徴**：左側が分解（トップダウン）、右側が統合（検証）。

### 6 つの認識側面

**意味**：問題を多面的に捉える 6 つの観点。

1. 技術的（どう動くか）
2. 人的（誰が関わるか）
3. 組織的（どの組織か）
4. 経済的（コスト・便益）
5. 政治的（権力・利害）
6. 倫理的（何が正しいか）

## 2. 構造化手法（ISM・DEMATEL）

### 構造化手法 ⭐

**意味**：要素間の関係を行列で表現し、システムの構造を可視化する手法群。

**代表手法**：ISM、DEMATEL、Cognitive Map、DSM、MDM

**用途**：複雑な要因系の整理、合意形成、根本要因の発見。

**試験での出方**：2023 で 15/50 点。

🔗 ISM、DEMATEL、一対比較

### 一対比較 ⭐

**意味**：要素を 2 つずつ取り出して関係を評価する手法。

**用途**：
- ISM：「i は j に影響するか」（0/1）
- DEMATEL：「i から j への影響の強さ」（0〜4）
- AHP：「i と j の相対重要度」（1〜9）

**比較数**：n 要素で n(n−1)/2 回。

🔗 ISM、DEMATEL、AHP

### バイナリ行列 ⭐

**意味**：要素が 0 / 1 のみの行列。ISM の隣接行列がこれ。

**例**：
```
   v1 v2 v3
v1  0  1  0
v2  0  0  1
v3  1  0  0
```

🔗 隣接行列、ブール代数演算則

### ブール代数演算則 ⭐

**意味**：0 / 1 の論理演算（AND, OR）。可到達行列の計算で使う。

**規則**：
- 0 + 0 = 0、0 + 1 = 1、1 + 1 = 1（OR）
- 0 × 0 = 0、0 × 1 = 0、1 × 1 = 1（AND）

**用途**：M = A + I の冪乗で、行列の積もブール演算で計算する。

🔗 可到達行列、バイナリ行列

### 隣接行列 ⭐

**意味**：a_ij = 1 で i → j に有向エッジがある行列。

**性質**：
- 有向グラフは非対称
- 無向グラフは対称
- A^k の (i, j) 成分 = i から j への長さ k の経路数

🔗 可到達行列、バイナリ行列

### 推移律 ⭐

**意味**：iRj かつ jRk なら iRk が成り立つ性質。ISM の前提。

**式**：(iRj) ∧ (jRk) ⇒ (iRk)

**反例**：
- 友人関係：A と B が友人、B と C が友人でも、A と C は友人とは限らない
- じゃんけん：グー > チョキ > パー だが、グー < パー
- 相性：A と B が相性◎、B と C が相性◎でも、A と C は相性 ×

**試験での出方**：「ISM の前提として推移律を仮定する」「推移律が成り立たない例を挙げよ」

🔗 ISM、可到達行列

### 収束 ⭐

**意味**：M = A + I の冪を繰り返すと、ある回数で M^k = M^(k+1) になる現象。

**性質**：n ノードのグラフでは **n − 1 回** で収束（最長単純経路の長さ）。

**確認方法**：M, M², M³ ... と計算して変化が止まる回数で判定。

🔗 可到達行列

### 可到達行列 ⭐

**意味**：i から j に何らかの経路で到達可能かを示す行列。

**式**：M^(n−1)（M = A + I の n − 1 乗）

**意味**：M^(n−1) の (i, j) 成分が 1 なら、i から j に長さ 0〜n−1 の経路で到達可能。

**用途**：ISM の階層抽出の元データ。

🔗 隣接行列、骨格行列、推移律

### 骨格行列（スケルトン行列）⭐

**意味**：A → B → C のとき派生する A → C のような**冗長な推移関係**を除いた**最小構造関係**。

**用途**：ISM 階層図の描画。冗長を除くと階層構造が見やすくなる。

🔗 可到達行列、ブロック三角化

### ブロック三角化

**意味**：強連結成分（互いに到達可能なノード群）をまとめ、階層順に並べ替える前処理。

**用途**：ISM の階層化前に、循環構造を 1 ブロックにまとめる。

### R 集合（到達可能集合）⭐

**意味**：i から到達できる要素の集合（i 自身も含む）。「i の下流」。

**式**：R(i) = {j | M^(n−1)_{ij} = 1}

### A 集合（先行集合）⭐

**意味**：i に到達できる要素の集合（i 自身も含む）。「i の上流」。

**式**：A(i) = {j | M^(n−1)_{ji} = 1}

### R ∩ A = R ⭐

**意味**：i が ISM の階層で**最上位レベル**にある条件。

**意味の解釈**：「i から到達できる要素はすべて i にも到達できる」つまり「**i から下流に出られない＝もう下に要素がない＝最上位**」。

**アルゴリズム**：
1. R(i) ∩ A(i) = R(i) を満たす i を最上位レベルとして抽出
2. 抽出した要素を除いて反復
3. すべての要素が階層化されるまで繰り返す

**試験での出方**：「R ∩ A = R の意味を説明せよ」

### ISM 法 ⭐

**意味**：要素間の推移的依存関係から階層構造を抽出する手法。Warfield 1973。

**手順**：
1. 要素抽出
2. 一対比較で隣接行列 A
3. M = A + I の冪で可到達行列
4. R ∩ A = R で階層抽出
5. 骨格行列で階層図

**用途**：政策目標体系の構造化、組織課題の階層化。

🔗 推移律、可到達行列、R 集合、A 集合、骨格行列

### DEMATEL 法 ⭐

**意味**：要素間の影響を 0〜4 で評価し、直接影響と間接影響を含めた総合影響を計算する手法。Battelle Geneva 1972。

**手順**：
1. 直接影響行列 X（0〜4 の評価）
2. 正規化：N = X / 最大行和
3. 総合影響：T = N(I − N)⁻¹ = N + N² + N³ + …
4. D（行和）= 発信力、R（列和）= 被影響力
5. D − R で原因系/結果系、D + R で中心性

**試験での出方**：用語論述で「DEMATEL を活用する方法を述べよ」が出る。**数値計算は過去 5 年で出題なし**。

🔗 直接影響行列、間接的影響行列、影響度、関連度

### 直接影響行列 X ⭐

**意味**：DEMATEL の入力。i から j への直接影響を 0〜4 で評価。

**評価尺度**：0=影響なし、1=弱、2=中、3=強、4=非常に強い。

### 正規化行列 N

**意味**：直接影響行列 X を最大行和で割って [0, 1] に収めた行列。

**式**：N = X / max(各行の行和)

**理由**：T = N + N² + N³ + … の級数を**収束させる**ため。

### 間接的影響行列 ⭐

**意味**：N², N³, ... の冪和で得られる、間接経由の影響を表す行列。

**例**：N² は「i → 中継 → j」の 2 段階間接影響。

🔗 総合影響行列

### 総合影響行列 T ⭐

**意味**：直接影響と間接影響の総和。

**式**：T = N(I − N)⁻¹ = N + N² + N³ + …

**N が冪零なら**：(I − N)⁻¹ = I + N + N² で有限和。

🔗 直接影響行列、間接的影響行列、影響度、関連度

### 影響度（D − R）⭐

**意味**：要素 i の因果性。発信力 D（行和）− 被影響力 R（列和）。

**判定**：
- **D − R > 0 → 原因系**（介入すべき根本要素）
- **D − R < 0 → 結果系**（症状にあたる要素）

**試験での出方**：「DEMATEL でどの要素が原因系か判別する方法」

🔗 D、R、関連度

### 関連度（D + R）⭐

**意味**：要素 i のシステム内での中心性。発信力 + 被影響力。

**判定**：D + R が大きいほど、システムの中心的要素。

🔗 D、R、影響度

### D（発信力）⭐

**意味**：要素 i が他のすべての要素に与える総影響量。

**式**：D_i = Σ_j t_{ij}（T の i 行の和）

### R（被影響力）⭐

**意味**：要素 i が他のすべての要素から受ける総影響量。

**式**：R_j = Σ_i t_{ij}（T の j 列の和）

### DSM（Design Structure Matrix）⭐

**意味**：設計要素間の依存関係を行列化する手法。製品設計で使う。

**特徴**：要素は同じドメイン（例：すべて設計コンポーネント）。

**用途**：製品アーキテクチャ分析、開発工程の並列化。

**試験での出方**：2022 で用語論述に登場。

### MDM（Multiple Domain Matrix）⭐

**意味**：複数ドメイン間の関係を扱う行列。組織×製品×プロセス等の統合。

**特徴**：DSM の拡張。異なるドメインの要素の関係を捉える。

**試験での出方**：2022, 2023 で用語論述に登場。

## 3. 因果ループ・SD

### Cognitive Map（CM）⭐

**意味**：要素間の因果関係を**符号付き有向グラフ**で表現する手法。Axelrod 1976。

**特徴**：
- **+** リンク：原因が増えれば結果も増える（正の因果）
- **−** リンク：原因が増えれば結果が減る（負の因果）

**DEMATEL との違い**：CM は**符号のみ**、DEMATEL は**強さ（0〜4 非負）のみ**。

**FCM**（Fuzzy Cognitive Map、Kosko 1986）：重みを [−1, +1] のファジィ値にした拡張。

### 因果ループ図（CLD）⭐

**意味**：システム内の要素間の因果と符号を可視化した図。SD の質的記法。

**記号**：
- **S（Same）**：同方向変化（A 増 → B 増）
- **O（Opposite）**：逆方向変化（A 増 → B 減）

**ループの種類**：
- **強化ループ R**：O が偶数個。自己強化・発散
- **バランスループ B**：O が奇数個。均衡・収束

**試験での出方**：2022（商品の魅力・売上・原資のループ）、2023 で必答 50 点。

🔗 S/O、強化ループ、バランスループ、システムダイナミクス

### S（Same）リンク

**意味**：同方向変化のリンク。A が増えれば B も増える、A が減れば B も減る。

**例**：「勉強時間↑ → 学力↑」

### O（Opposite）リンク

**意味**：逆方向変化のリンク。A が増えれば B が減る、A が減れば B が増える。

**例**：「価格↑ → 需要↓」

### 強化ループ R ⭐

**意味**：自己強化・発散するループ。一度動き出すと暴走する。

**判定**：ループ内の **O が偶数個**。

**例**：「貯金↑ → 利息↑ → 貯金↑」（O が 0 個、複利の好循環）

**試験での出方**：「次のループが R か B か判定せよ」

### バランスループ B ⭐

**意味**：均衡・収束するループ。安定化させる。

**判定**：ループ内の **O が奇数個**。

**例**：「体温↑ → 発汗↑ → 体温↓」（O が 1 個）

### 8 つのシステム原型

**意味**：SD でよく出る典型パターン。Senge が整理。

1. **Fix That Fails**（応急処置の失敗）：B（即効）+ R（副作用）
2. **Shifting the Burden**（負担の転嫁）
3. **Limits to Growth**（成長の限界）
4. **Tragedy of the Commons**（共有地の悲劇）
5. **Eroding Goals**（漂流する目標）
6. **Success to the Successful**（成功には成功を）
7. **Escalation**（エスカレート）
8. **Growth and Underinvestment**（成長と投資不足）

### システムダイナミクス（SD）⭐

**意味**：因果ループを定量化し、時間発展をシミュレーションする手法。Forrester 1961。

**構成要素**：ストック、フロー、コンバータ

**用途**：政策実験、長期予測、シナリオ分析。

**代表例**：World3（Limits to Growth）、Urban Dynamics。

**試験での出方**：2022 で用語論述、2023 で 10/50 点。

🔗 ストック、フロー、コンバータ、因果ループ図

### ストック ⭐

**意味**：時点で測れる蓄積量（状態変数）。積分量。

**例**：人口、貯金、CO₂ 濃度、在庫、蓄電量

**性質**：過去の蓄積を持つため、システムに**慣性・遅延**を生む。

🔗 フロー、コンバータ

### フロー ⭐

**意味**：単位時間あたりの増減量。微分量。ストックを変化させる。

**例**：出生率、入金、排出率、出荷率、発電量

**式**：ストックの時間微分 = 流入フロー − 流出フロー

🔗 ストック、コンバータ

### コンバータ

**意味**：SD モデルの補助変数。フローやストックを計算する中間値。

**例**：人口モデルの出生率係数、価格弾性。

## 4. ファジィ理論

### ファジィ概念 ⭐

**意味**：境界が曖昧な概念。「重い」「軽い」「高い」「若い」「暑い」など。

**特徴**：白黒では判別できない。**程度**の問題。

**例**：「高齢」は何歳から？「速い」は時速何キロから？

🔗 ファジィ集合、メンバーシップ関数

### ファジィ集合 ⭐

**意味**：要素の所属の度合いを [0, 1] で表す集合。Zadeh 1965。

**特徴**：通常の集合（属する／属さないの二択）を一般化したもの。

**形式**：要素 x のメンバーシップ度 μ_A(x) ∈ [0, 1]

🔗 メンバーシップ関数、ファジィ数、ファジィ関係

### ファジィ数 ⭐

**意味**：ファジィ集合で表される数。「だいたい 5」「約 10」「ほぼ 100」など。

**用途**：曖昧な数値情報を扱う。例：「商品の価格は約 100 円」。

🔗 ファジィ集合

### ファジィ関係 ⭐

**意味**：要素間の関係をファジィ集合で表したもの。

**例**：「A は B にやや似ている」「X は Y にかなり関係がある」

**用途**：ファジィ推論で多変数の関係を扱う。

### メンバーシップ関数 μ_A(x) ⭐

**意味**：x が集合 A に属する度合いを [0, 1] で表す関数。

**範囲**：
- μ = 1：完全に所属
- μ = 0：完全に非所属
- 中間値：部分的所属

**形状**：三角形、台形、シグモイドなどが典型。

**例**：「若い」のメンバーシップ
- μ(20歳) = 1.0、μ(40歳) = 0.5、μ(60歳) = 0.1

**試験での出方**：「『重い』『軽い』のメンバーシップ関数を図示せよ」（2022, 2023）

### t-norm（ファジィ AND）

**意味**：ファジィ集合の AND 演算。

**典型**：μ_A AND μ_B = **min(μ_A, μ_B)**

**代替**：積 μ_A · μ_B も可。

### s-norm（ファジィ OR）

**意味**：ファジィ集合の OR 演算。

**典型**：μ_A OR μ_B = **max(μ_A, μ_B)**

**代替**：確率和 μ_A + μ_B − μ_A · μ_B も可。

### NOT（ファジィ否定）

**式**：NOT μ_A = **1 − μ_A**

### ファジィ推論

**意味**：ファジィルールを使った推論。「もし暑いなら冷房を強くせよ」のようなルール群から、明確な出力を得る。

**4 ステップ**：
1. **ファジィ化**：明確な入力（例：温度 28℃）→ メンバーシップ度（「やや暑い」0.7）
2. **ルール適用**：複数の IF-THEN ルール
3. **合成**：結果を集約
4. **デファジィ化**：ファジィ結果 → 明確な値（重心法など）

**応用**：エアコン自動制御、洗濯機、新幹線自動運転。

### FCM（Fuzzy Cognitive Map）

**意味**：Cognitive Map の拡張。重みを [−1, +1] のファジィ値にしたもの。Kosko 1986。

**位置づけ**：CM（符号のみ）と DEMATEL（強さのみ）の中間。**符号 + 強さ**両方を扱う。

### 確率 vs ファジィ ⭐⭐

| | 確率 | ファジィ |
|---|------|----------|
| 対象 | 事象の**発生**の不確実性 | 概念の**境界の曖昧さ** |
| 例 | 雨が降る確率 60% | 「高齢」の度合い 0.7 |
| 数学的基盤 | Kolmogorov の確率公理 | メンバーシップ関数 |
| 解釈 | 何度も繰り返したら 60% で起きる | 「高齢」概念に 70% 属する |

**試験での出方**：「確率とファジィの違いを述べよ」が頻出。

## 5. AHP

### AHP（Analytic Hierarchy Process）⭐

**意味**：多基準意思決定の代表手法。Saaty 1980。

**目的**：複数の評価基準のもとで、複数の代替案から 1 つを選ぶ。

**手順**：
1. 階層構造（目標 → 評価基準 → 代替案）の構築
2. 一対比較（1〜9 尺度）
3. 重みの算出（固有ベクトル法または幾何平均法）
4. 整合性 CR の検証（CR < 0.1 で OK）
5. 総合優先度の計算

**応用**：新車購入、公共事業の候補地選定、人事評価。

🔗 一対比較、Saaty 尺度、CR

### Saaty の 1〜9 尺度

**意味**：一対比較で使う相対重要度の尺度。

| 値 | 意味 |
|----|------|
| 1 | 同等 |
| 3 | やや優 |
| 5 | 優 |
| 7 | かなり優 |
| 9 | 極めて優 |
| 2, 4, 6, 8 | 中間値 |

**逆数**：a_ji = 1 / a_ij（i が j より「やや優」なら、j は i より「やや劣」）。

### 整合性比 CR（Consistency Ratio）

**意味**：判断の一貫性を検証する指標。

**式**：CR = CI / RI（CI: 整合性指標、RI: ランダム指標）

**判定**：**CR < 0.1** で「一貫した判断」とみなす。超えたら判断を見直す。

**用途**：人間の循環矛盾（A>B>C>A など）を検出する。

### ランキング逆転問題

**意味**：代替案を 1 つ追加すると、既存の順位が変わることがある AHP の限界。

**例**：A > B > C の順位だったのに、D を追加すると B > A になる。

## 6. 確率モデル基礎

### 試行

**意味**：結果が偶然に決まる実験。

**例**：コイン投げ、サイコロ振り、客の来店。

### 標本空間 Ω

**意味**：すべての結果の集合。

**例**：コイン投げなら Ω = {表, 裏}。

### 事象

**意味**：Ω の部分集合。

**例**：サイコロで「偶数の目が出る」事象 = {2, 4, 6}。

### 確率公理（Kolmogorov）

**意味**：確率論の数学的基礎を 3 つの公理にまとめたもの。

**3 つの公理**：
1. **非負性**：0 ≤ P(A) ≤ 1
2. **全体の確率は 1**：P(Ω) = 1
3. **可算加法性**：互いに排反な事象 A_i に対し P(∪A_i) = Σ P(A_i)

### 条件付き確率

**意味**：B が起きたとき A が起きる確率。

**式**：P(A | B) = P(A ∩ B) / P(B)

### ベイズの定理

**式**：P(A | B) = P(B | A) P(A) / P(B)

**応用**：医療診断、スパムフィルタ、Naive Bayes 分類器。

### 確率モデル

**意味**：確率的なルールで事象をモデル化したもの。

### 確率過程 ⭐

**意味**：状態が**時間とともに変化する**確率変数族 {X_t}。

**例**：マルコフ連鎖、ランダムウォーク、ブラウン運動。

🔗 マルコフ性

### マルコフ性 ⭐⭐

**意味**：次の状態が現在の状態だけで決まり、過去の履歴に依存しない性質。

**式**：

$$P(X_{n+1} = j | X_n = i, X_{n-1}, \ldots, X_0) = P(X_{n+1} = j | X_n = i)$$

**言葉**：「**過去の履歴を忘れて、現在の状態のみで未来が決まる**」

**意義**：推移行列 P だけで未来を予測できる → 解析がシンプル。

**例**：天気の遷移（今日が晴れなら明日の確率分布が決まる）、ランダムウォーク、ギャンブラーの破産。

**試験での出方**：用語論述、誤り指摘で頻出。

🔗 推移確率、確率過程、定常分布

## 7. 確率分布

### 分布関数 F(x)

**意味**：累積分布関数（CDF）。F(x) = P(X ≤ x)。

**性質**：
1. 単調非減少
2. 右連続
3. F(−∞) = 0, F(∞) = 1

**用途**：離散・連続どちらの確率変数にも定義可能。

### 確率質量関数 p(x)

**意味**：離散型確率変数の各値の確率。p(x) = P(X = x)。

### 確率密度関数 f(x)

**意味**：連続型確率変数の密度。f(x) ≥ 0、∫f = 1。

**注意**：連続型では P(X = x) = 0。確率は区間積分で。

### 期待値 E[X]

**式**：
- 離散：E[X] = Σ x_i p(x_i)
- 連続：E[X] = ∫ x f(x) dx

**性質**：線形性 E[aX + b] = aE[X] + b

### 分散 V[X]

**式**：V[X] = E[(X − E[X])²] = E[X²] − (E[X])²

### ベルヌーイ分布

**意味**：1 回の成功 / 失敗（成功確率 p）。

**期待値**：p、**分散**：p(1−p)

### 二項分布 B(n, p)

**意味**：n 回独立試行の成功数。

**式**：P(X = k) = C(n, k) p^k (1−p)^(n−k)

**期待値**：np、**分散**：np(1−p)

### ポアソン分布

**意味**：単位時間の稀な事象数。

**式**：P(X = k) = e^(−λ) λ^k / k!

**期待値・分散**：λ

### 幾何分布

**意味**：初成功までの試行回数。

**期待値**：1/p、**分散**：(1−p)/p²

### 指数分布

**意味**：事象間隔（待ち時間）。

**式**：f(x) = λe^(−λx)（x ≥ 0）

**期待値**：1/λ、**分散**：1/λ²

**特徴**：**無記憶性**を持つ。

🔗 無記憶性

### 無記憶性 ⭐

**意味**：指数分布の特別な性質。

**式**：P(X > s + t | X > s) = P(X > t)

**言葉**：「これまで s 時間待ったという情報が、残り待ち時間の分布に影響しない」

**例**：バスを 10 分待ったが来ない → さらに 5 分待つ確率は、待ち始めの瞬間に 5 分待つ確率と同じ。

**意義**：マルコフ性と概念的に同型。M/M/1 待ち行列の理論的基盤。

### 正規分布 N(μ, σ²)

**意味**：測定誤差、自然現象の代表分布。

**式**：f(x) = (1/√(2πσ²)) exp(−(x−μ)²/(2σ²))

**期待値**：μ、**分散**：σ²

### 中心極限定理

**意味**：独立同分布の和は n が大きいと正規分布に近づく。

**式**：(X̄ − μ) / (σ/√n) → N(0, 1)

**意義**：「個々の分布が何であろうと、たくさん足せば正規分布」が成り立つ。統計の基礎。

### モンテカルロ法

**意味**：乱数サンプルで期待値・確率を近似する方法。

**誤差**：1/√N のオーダー（大数の法則）。

**応用**：高次元積分、リスク評価、複雑系シミュレーション。

### 大数の法則

**意味**：サンプル数を無限にすると、標本平均が真の期待値に収束する。

**式**：X̄_n → μ（n → ∞）

## 8. マルコフ連鎖・待ち行列

### マルコフ連鎖 ⭐

**意味**：マルコフ性を持つ離散時間確率過程。

**構成**：状態空間 S = {1, 2, ..., m}、推移確率行列 P。

### 推移確率 p_ij ⭐

**意味**：状態 i から状態 j への 1 ステップ遷移確率。

**式**：p_ij = P(X_{n+1} = j | X_n = i)

### 推移確率行列 P ⭐

**意味**：(p_ij) を並べた行列。

**性質**：各行の和 = 1（必ずどこかに遷移するから）。

### l ステップ推移確率

**式**：p_ij^(l) = (P^l)_ij

**意味**：l ステップ後に状態 i から j にいる確率。

### Chapman-Kolmogorov 方程式

**式**：p_ij^(l+n) = Σ_k p_ik^(l) p_kj^(n)

**意味**：「l ステップで k に行き、その後 n ステップで j に行く確率の総和」。

### 定常分布 π ⭐⭐

**意味**：「1 ステップ進んでも分布が変わらない」分布。

**式**：πP = π かつ Σπ_i = 1

**意義**：長時間後の状態分布（既約 + 非周期的なら極限分布と一致）。

**計算例**：P = [[0.9, 0.1], [0.2, 0.8]] → π = (2/3, 1/3)

**試験での出方**：「定常分布を求めよ」「走行割合を求めよ」

🔗 マルコフ連鎖、推移確率行列

### 極限分布

**意味**：n → ∞ で p^(0) P^n が収束する分布。

**条件**：既約 + 非周期的 + 再帰的 で、定常分布と一致。

### 既約

**意味**：全状態が相互に到達可能。

### 再帰的

**意味**：有限時間で戻ってこれる。

### 過渡的

**意味**：戻らない確率が正。

### 非周期的

**意味**：周期が 1（戻り間隔がバラバラ）。

### 吸収状態

**意味**：p_ii = 1 で、そこから出られない状態。

**例**：ギャンブラーの破産問題で、所持金 0 や N の状態。

### 待ち行列理論

**意味**：到着・サービス過程のモデル化。Erlang 1909（電話交換機の設計）。

**応用**：銀行 ATM、コールセンター、Web サーバー、製造ライン。

### ケンドール記法 A/B/c

- A：到着過程
- B：サービス過程
- c：サーバ数

**例**：
- M/M/1：ポアソン到着・指数サービス・1 サーバ
- M/M/c：c 台サーバ
- M/G/1：サービスが一般分布

### M/M/1 ⭐

**意味**：最も基本の待ち行列モデル。

- **M（Markov）**：到着がポアソン過程（指数分布の到着間隔）
- **M**：サービス時間も指数分布
- **1**：サーバ 1 台

**試験での出方**：2022, 2023 で「店内で待っている客の期待値」が出題。

### 利用率 ρ ⭐⭐

**意味**：サーバが忙しい時間の割合。

**式**：ρ = λ/μ

- λ：到着率
- μ：サービス率

**安定条件**：ρ < 1（ρ ≥ 1 で発散）

### 安定条件

**意味**：M/M/1 が安定（待ち行列が発散しない）になる条件。

**式**：ρ < 1

**意義**：ρ ≥ 1 だと到着が処理を上回り、無限に列が伸びる。

### リトルの法則 ⭐

**式**：L = λW

- L：系内平均人数
- λ：到着率
- W：平均滞在時間

**意義**：分布によらず常に成立する普遍法則。

### M/M/1 公式集

| 量 | 式 |
|----|-----|
| 利用率 | ρ = λ/μ |
| 系内平均人数 L | ρ/(1−ρ) |
| 平均滞在時間 W | 1/(μ−λ) |
| 平均待ち行列長 L_q | ρ²/(1−ρ) |
| 平均待ち時間 W_q | ρ/(μ−λ) |

**注意**：「店内で待っている人」は L_q（窓口の人を除く）、「店内の総数」は L（窓口の人も含む）。

### 走行割合（マルコフ定常分布の応用）⭐

**過去問例**（2022, 2023）：

「トラックの後ろを走る車は 4 台中 3 台が乗用車、乗用車の後ろを走る車は 5 台中 1 台がトラック」

→ 推移行列：P = [[1/4, 3/4], [1/5, 4/5]]

→ 定常分布：π_T = 4/19、π_P = 15/19

→ **トラック : 乗用車 = 4 : 15**

### 2 次元ランダムウォーク ⭐

**過去問例**（2023）：

「xy 平面の原点から 4 方向に確率 1/4 で動く。2n 回後に原点に戻る確率は？」

**答え**：

$$
P_{2n}(0, 0) = \binom{2n}{n}^2 \left(\frac{1}{4}\right)^{2n}
$$

スターリング近似で：

$$
\approx \frac{1}{\pi n}
$$

→ 2 次元では原点に戻る確率は n に反比例して減衰する（再帰的）。

### スターリングの公式

**式**：n! ≈ √(2πn) (n/e)^n

**用途**：階乗の漸近評価。2 項係数の評価などで使う。

**応用**：C(2n, n) ≈ 4^n / √(πn)

## 9. オートマトン・セルオートマトン

### 有限オートマトン ⭐

**意味**：5 つ組 (Q, Σ, δ, q_0, F) で定義される離散系。

- **Q**：状態集合
- **Σ**：入力アルファベット
- **δ**：遷移関数 Q × Σ → Q
- **q_0**：初期状態
- **F ⊆ Q**：受理状態

**用途**：文法解析、字句解析、コンパイラ、正規表現マッチング。

🔗 決定性有限オートマトン、非決定性有限オートマトン

### 決定性有限オートマトン（DFA）⭐

**意味**：各状態と入力に対し、遷移先が**一意に決まる**オートマトン。

### 非決定性有限オートマトン（NFA）⭐

**意味**：各状態と入力に対し、遷移先が**複数あり得る**オートマトン。ε 遷移も含む。

**性質**：DFA と等価な記述能力を持つ（部分集合構成法で変換可能）。

### 部分集合構成法（NFA → DFA 変換）⭐

**意味**：NFA を等価な DFA に変換するアルゴリズム。

**手順**：
1. 初期状態：{q_0}（NFA の初期状態を含む単集合）
2. 各既存状態 S と入力 a について δ'(S, a) = ∪_{q ∈ S} δ(q, a) を計算
3. 新規状態なら追加
4. 受理状態：F' = {S | S ∩ F ≠ ∅}

**試験での出方**：2016, 2022, 2023, 2025 で出題。状態遷移表で答える。

### 受理言語

**意味**：オートマトンが受理する文字列の集合。

**表現**：正規表現で書く。

**例**：
- **a (bb)\***：1 個の a の後に偶数個の b
- **a\* b\***：0 個以上の a の後に 0 個以上の b
- **(ab)\***：a と b の交互列

### 正規表現

**記法**：
- **a**：1 文字 a
- **ab**：a の後に b
- **a***：a を 0 回以上繰り返し
- **a+**：a を 1 回以上繰り返し（= aa\*）
- **a|b**：a または b
- **ε**：空文字列

### セルオートマトン（CA）⭐

**意味**：4 つ組 (L, S, N, f) で定義。格子状の多数セルが同じ局所ルールで同時更新される系。

- **L**：格子（1D, 2D, ...）
- **S**：各セルの状態集合
- **N**：近傍関係
- **f**：更新関数

**特徴**：単純なルールから複雑挙動が創発。

### von Neumann 近傍

**意味**：2D の近傍。**上下左右の 4 マス**。

### Moore 近傍

**意味**：2D の近傍。**周囲 8 マス**（斜めも含む）。

### Game of Life（Conway 1970）⭐

**意味**：2D CA の最有名例。各セルは「生」または「死」の 2 状態。

**ルール**（Moore 近傍）：
- **生**のセル：隣接生セルが **2 か 3** で生存（それ以外は死亡）
- **死**のセル：隣接生セルが **ちょうど 3** で誕生

**創発**：静物、振動子、グライダー、グライダー銃、任意の論理回路（**チューリング完全**）。

### Wolfram コード ⭐

**意味**：1D CA のルール番号。Stephen Wolfram の表記。

**例**：ルール 22

| 入力 (左, 中央, 右) | 次の状態 |
|--------------------|----------|
| 111 | 0 |
| 110 | 0 |
| 101 | 0 |
| 100 | 1 |
| 011 | 0 |
| 010 | 1 |
| 001 | 1 |
| 000 | 0 |

2 進数で 00010110 = 22 で表す。

**試験での出方**：2022 でルール 22 の状態遷移図が出題。

### Wolfram の CA 分類

| クラス | 挙動 |
|--------|------|
| Class I | 均一に収束 |
| Class II | 周期パターン |
| Class III | カオス（ランダムに見える） |
| Class IV | 複雑構造（チューリング完全） |

### サンドパイル・モデル ⭐

**意味**：自己組織化臨界（SOC）の典型的 CA モデル。Bak, Tang, Wiesenfeld 1987。

**更新規則**：
1. ランダムに 1 セルを選び、値を +1
2. 値が 4 以上なら、4 減らし、4 近傍に +1 ずつ
3. 連鎖崩壊を 3 以下になるまで繰り返す

**結果**：崩壊規模が**べき乗分布**になる → SOC の典型例。

## 10. エージェント・マルチエージェント

### エージェント ⭐

**意味**：環境を**知覚**し、自律的に**行動**する個体。

**特徴**：
- 自分の状態を持つ
- 環境を観察できる
- 目標や効用関数を持つ（場合がある）
- 学習することもある（場合がある）

🔗 反応性、自発性、合理性、完全性、MAS

### 反応性（reactivity）⭐

**意味**：エージェントが環境の変化を知覚し、**即座に反応**する能力。

**特徴**：事前のルールに従って動く。内部モデルや目標を持たなくてもよい。

**例**：サーモスタットが温度変化を感知して冷暖房を切り替える。

**試験での出方**：2022, 2023, 2025 で「反応性と自発性・先見性の違い」が出題。

### 自発性・先見性（proactiveness）⭐

**意味**：エージェントが**自分の目標に向かって主体的に行動**する能力。未来を予測して計画的に動く。

**例**：自動運転車が渋滞を予測してルート変更する。

**反応性との対比**：反応性は受動的、自発性は能動的。

### 合理性（rationality）⭐

**意味**：エージェントが**与えられた情報と目標に基づいて、効用を最大化する選択をする**性質。

**試験での出方**：2023, 2025 で「合理性と完全性の違い」が出題。

🔗 完全性、効用ベース

### 完全性（completeness）⭐

**意味**：エージェントが**環境のすべての情報を持ち、すべての選択肢を考慮できる**性質。

**現実**：完全性は不可能で、**限定合理性**（限られた情報での意思決定）が普通。

**例**：チェス AI は計算範囲内では合理的だが、すべての手を読み切れないため完全ではない。

### 単純条件反射エージェント ⭐

**意味**：現在の知覚に対して IF-THEN ルールで行動するエージェント。内部状態なし。

**特徴**：完全観測可能な環境のみで適用可能。

**例**：サーモスタット。

**試験での出方**：2016, 2025 で「モデルベースとの違い」が出題。

### モデルベース条件反射エージェント ⭐

**意味**：**内部に世界モデル**を持ち、それを更新しながら行動するエージェント。

**特徴**：部分観測可能な環境にも適用可能。見えない部分を推測できる。

**例**：自動運転車（地図情報を内部に持つ）。

### 目標ベースエージェント

**意味**：明示的な目標を持ち、その達成手段を選ぶエージェント。

**例**：経路探索（目的地に到着する経路を計画）。

### 効用ベースエージェント

**意味**：効用関数の期待値を最大化する合理的選択をするエージェント。

**例**：投資ボット（利益とリスクを天秤にかけて選択）。

### 学習エージェント

**意味**：経験から自分を改善するエージェント。

**例**：強化学習（試行錯誤で報酬を最大化）。

### マルチエージェントシステム（MAS）⭐

**意味**：多数のエージェントが相互作用する系。

**特徴**：
- **集中制御がない**
- **局所相互作用**
- **創発**：個々の単純ルールから集団レベルで予測困難な性質が生まれる

🔗 創発、Schelling 分居モデル、boids

### 創発（emergence）⭐

**意味**：局所ルールから**集団レベルで予測困難な大域的性質**が生じる現象。

**代表例**：
- **Schelling の分居モデル**：弱い人種選好 → 強い分居
- **Reynolds の boids**：3 つの局所ルール → 鳥群の飛行
- **アリのフェロモン**：局所判断 → 最短経路発見
- **市場参加者**：個別売買 → 価格動学・暴落

### Schelling の分居モデル

**意味**：弱い人種選好（周囲の少なくとも 30% が同じ人種であってほしい）から、強い分居が生まれることを示したモデル。1971。

**教訓**：個人レベルの差別意識が弱くても、社会レベルでは強い分離が出現しうる。創発の典型例。

### Reynolds の boids

**意味**：3 つの局所ルールで鳥の群れ飛行を再現するモデル。

**3 ルール**：
1. **分離**：近くの個体から離れる
2. **整列**：周りの個体と方向を揃える
3. **凝集**：群れの中心に近づく

### 静的タスク環境

**意味**：エージェントの行動の影響が**その場限り**で、未来に影響しない環境。

**例**：チェス（盤面が固定）。

### 動的タスク環境

**意味**：エージェントが行動を選ぶ間に環境が変わる環境。

### 競争的マルチエージェント環境

**意味**：複数のエージェントが**互いの利得を最大化しようとして対立する**環境。

**例**：オークション、対戦ゲーム、市場での競合企業。

### 協調的マルチエージェント環境

**意味**：エージェント同士が協力して共通目標を達成する環境。

**例**：チームスポーツ、共同作業。

### PEAS 表現

**意味**：エージェントを設計するときの 4 要素。

- **P（Performance、性能指標）**：何を最大化するか
- **E（Environment、環境）**：エージェントが置かれる状況
- **A（Actuator、アクチュエータ）**：エージェントが環境に働きかける手段
- **S（Sensor、センサ）**：環境から情報を得る手段

**試験での出方**：2015, 2016 で「ニュース推薦エージェント」「スマホカメラエージェント」の PEAS 設計が出題。

## 11. 複雑系・SOC

### 複雑系（complex system）⭐

**意味**：局所相互作用から大域挙動が生まれるが、全体の振る舞いが個々の要素から予測困難な系。

**代表モデル**：セルオートマトン、マルチエージェント。

**特徴**：べき乗則、フラクタル、1/f ゆらぎ、スケール不変。

🔗 SOC、べき乗則、スケール不変

### 複雑適応系（CAS）

**意味**：構成要素が環境に適応して振る舞いを変える複雑系。

**例**：生態系、市場、免疫系。

### 自己組織化臨界（SOC）⭐

**意味**：外部調整なしに、系が自分で臨界状態を保つ現象。Bak, Tang, Wiesenfeld 1987。

**典型例**：サンドパイル・モデル（崩壊規模がべき乗分布）。

**現実例**：
- 地震のサイズ vs 頻度（Gutenberg-Richter 則）
- 山火事の規模
- 金融市場の暴落
- 神経活動（ニューロン雪崩）

### べき乗則 ⭐

**意味**：頻度 N(x) がサイズ x のべき乗に反比例する関係。

**式**：N(x) ∝ x^(−τ)

**特徴**：両対数プロットで**直線**になる。

**現実例**：
- **Gutenberg-Richter 則**：地震
- **Zipf の法則**：都市規模 vs ランク
- **パレート分布**：所得分布

### スケール不変（スケールフリー）⭐⭐

**意味**：複雑系の特性で、**特定のスケール（時空間）に依存しない性質**。

**特徴**：
- べき乗則 N(x) ∝ x^(−τ) に従う
- 両対数プロットで直線
- 大規模事象も小規模と同じ法則で起きる

**現実例**：地震、所得分布、ネットワークの次数分布（Web リンク等）。

**意義**：正規分布前提のリスク評価では極端事象を過小評価する → **ブラック・スワン問題**。

**試験での出方**：2022, 2023, 2025 で 3 年連続出題。「複雑系の特性としてのスケール不変について説明せよ」

🔗 べき乗則、SOC、ブラック・スワン

### パレート分布

**意味**：べき乗則の連続版。確率密度関数 f(x) = α x_m^α / x^(α+1)。

**応用**：所得分布の高所得側（80:20 の法則）、ネットワークの次数分布。

### Zipf の法則

**意味**：都市規模 vs ランクなどに現れるべき乗則。

**例**：n 番目に大きい都市の人口 ∝ 1/n

### Gutenberg-Richter 則

**意味**：地震のマグニチュード（サイズ）と頻度の関係を表すべき乗則。

**式**：log N = a − bM（N: 頻度、M: マグニチュード）

### カオス

**意味**：決定論的だが**初期値に敏感**で予測困難な挙動。

**代表例**：
- **ローレンツ方程式**：気象モデル、「バタフライ効果」
- **ロジスティック写像**：x_{n+1} = r x_n (1 − x_n)

### フラクタル ⭐

**意味**：部分が全体に似る（**自己相似性**）図形。Mandelbrot。

**代表例**：マンデルブロ集合、海岸線、樹木の枝分かれ、雲。

🔗 フラクタル次元

### フラクタル次元 ⭐⭐

**意味**：自己相似フラクタルの次元。整数とは限らない。

**公式**：

$$D = \frac{\log N}{\log (1/r)} = \frac{\log N}{\log s}$$

- **N**：自己相似コピーの個数
- **r**：1 つのコピーの縮小率
- **s = 1/r**：全体を s 倍に拡大すると元と相似

**読み方**：「全体を s 倍に拡大すると、N 個の同じ形のコピーで埋められる」。

**試験での出方**：**5 年連続出題**（必出）。

🔗 シェルピンスキーのガスケット、カントール集合、メンガーのスポンジ

### シェルピンスキーのガスケット ⭐

**意味**：三角形を 3 つの小さな三角形で構成（中央を抜く）したフラクタル。

**次元**：
- N = 3（3 つのコピー）
- 各コピーは縦横 1/2 に縮小（s = 2）
- D = **log 3 / log 2 ≈ 1.585**

### カントール集合 ⭐

**意味**：線分を 3 等分し、両端の 2 つを残して中央を取り除く操作を無限回繰り返す。

**次元**：
- N = 2
- s = 3
- D = **log 2 / log 3 ≈ 0.631**

### メンガーのスポンジ ⭐

**意味**：立方体を 3×3×3 = 27 個に分け、面の中心 6 個と中央 1 個を取り除いて 20 個を残す。

**次元**：
- N = 20
- s = 3
- D = **log 20 / log 3 ≈ 2.727**

### コッホ曲線

**意味**：線分を 4 つの 1/3 縮小コピーで置き換える。

**次元**：D = log 4 / log 3 ≈ 1.262

### 平均場近似 ⭐

**意味**：多数の要素が相互作用する系で、**他の要素の影響を「平均値」で代表**させて各要素を独立に扱う近似手法。

**用途**：複雑な多体問題を扱いやすくする。

**例**：イジングモデルでスピン間の相互作用を平均磁場で近似する。

**試験での出方**：2022, 2025 で出題。

### 1/f ノイズ

**意味**：パワースペクトルが周波数に反比例するゆらぎ。

**例**：心拍、音楽、太陽黒点。

### ブラック・スワン

**意味**：正規分布前提では予測できない極端事象。べき乗則の世界では「100 年に 1 度」が思ったより頻繁に起きる。

**例**：リーマンショック、福島原発事故、コロナ。

## 12. 離散モデル化

### 離散化 ⭐

**意味**：連続現象を格子・差分で扱う手法。

**2 つの理由**：
1. **コンピュータは離散計算しかできない**：浮動小数点も有限精度
2. **連続体の PDE を解析的に解けない**：差分方程式に置き換えて数値解する

### 連続体近似

**意味**：ミクロ（原子・分子）から見れば粒子の集まりだが、マクロでは連続的に扱う。

**巨視変数**：
- 密度 ρ：微小体積内の粒子数 / 体積
- 速度 v：粒子の平均速度
- 圧力 p：粒子の運動量交換

### PDE（偏微分方程式）

**意味**：連続体現象の支配方程式。多変数関数の偏微分を含む。

**代表例**：Navier-Stokes、熱方程式、波動方程式。

### Navier-Stokes 方程式

**意味**：流体の運動方程式。

**式**：

$$
\frac{\partial \mathbf{v}}{\partial t} + (\mathbf{v}\cdot\nabla)\mathbf{v} = -\frac{1}{\rho}\nabla p + \frac{\mu}{\rho}\nabla^2 \mathbf{v} + \mathbf{F}
$$

**未解決問題**：解の存在と滑らかさはミレニアム懸賞問題の 1 つ。

### 有限差分法（FDM）

**意味**：微分を差分で近似する数値解法。

**用途**：単純な格子の CFD（流体数値解析）。

### 有限要素法（FEM）

**意味**：要素分割 + 基底関数で近似する数値解法。

**用途**：構造解析、複雑形状の問題。

### 有限体積法（FVM）

**意味**：体積積分の保存則を満たす数値解法。

**用途**：CFD（保存則を厳密に守りたい場合）。

### 粒子法（SPH 等）

**意味**：ラグランジュ粒子を追跡する数値解法。

**用途**：自由表面、衝突、爆発。

### CFL 条件

**意味**：双曲型 PDE の数値安定条件。時間刻み Δt は空間格子 Δx より十分小さく取る必要がある。

**式**：c Δt / Δx < 1（c: 波の速度）

### CFD（Computational Fluid Dynamics）

**意味**：流体数値解析。Navier-Stokes を離散化して解く。

### メゾモデル（メゾスコピックモデル）⭐

**意味**：ミクロとマクロの中間スケールのモデル。

**ミクロモデルに対する利点**：
- 計算量が少ない（個々の粒子を全て追跡しない）
- 大規模な系を扱える

**マクロモデルに対する利点**：
- 個体の異質性や離散性を保持できる
- 局所的な現象（揺らぎ、相転移）を扱える
- パラメータと実物理量の対応がつきやすい

**代表例**：粒子法、格子ボルツマン法、エージェントベースモデル。

**試験での出方**：2022, 2023 で「メゾモデルの利点」が出題。

### 格子ボルツマン法

**意味**：流体の運動を粒子の確率分布で扱うメゾスコピック手法。
