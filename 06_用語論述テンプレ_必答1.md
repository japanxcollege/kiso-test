| **システム要素** | システムを構成する個々の実体（人・物・情報など） |
| **相互関係** | 要素間の影響・因果・物質や情報の流れ |
| **システム思考** | フィードバックや相互依存を考慮する全体論的思考法。線形思考と対比 |
| **システム境界** | システム内外を区切る線。存在論的（実在）と認識論的（観点で変わる）の 2 種 |
| **システムの認識論** | システムを「どう見るか」「どこで区切るか」の議論 |
| **メンタルモデル** | 個人が世界を捉えるための内部表象。Senge の第 5 の規律 |
| **マン・マシン系** | 人間と機械が相互作用するシステム。人間も要素として扱う |

### 関連性

```
システム概念
  ├─ システム要素 + 相互関係 で構成される
  ├─ 全体として目的を持つ
  └─ システムの認識論 によって境界が決まる
       └─ システム境界（存在論的 vs 認識論的）

システム思考
  ├─ 線形思考の対極
  ├─ メンタルモデル に強く影響される
  ├─ 因果ループ図 で可視化
  └─ システムダイナミクス で定量化

マン・マシン系
  ├─ システム要素 として人間を含む
  ├─ システムの認識論 の典型例
  └─ 設計には システム思考 が必須
```

### 論述例（5 用語版・約 100 字 × 配点 10〜15 点）

> □システム概念□ は <u>複数の要素が相互に関係し、全体として目的を達成するもの</u>である。**=システムの認識論=** の立場では、<u>□システム境界□ の引き方は分析者の観点で変わる</u>。<u>=この境界設定が分析の出発点=</u> であり、要素や相互関係を整理する基礎となる。<u>=これらを総合的に扱う=</u> のが □システム思考□ で、<u>線形思考と異なり要素間のフィードバックを考慮する</u>。<u>=人間も要素として含む場合=</u> は □マン・マシン系□ となる。

## カテゴリ 2：構造化手法・ISM（第 2 回）

過去問頻出用語：**構造化手法・一対比較・バイナリ行列・ブール代数演算則・推移律・収束・可到達行列・骨格行列・隣接行列・直接影響行列・間接的影響行列・影響度/関連度・DSM・MDM**

### 用語の意味

| 用語 | 説明 |
|------|------|
| **構造化手法** | 要素間の関係を行列で表現し、構造を可視化する手法群（ISM, DEMATEL 等） |
| **一対比較** | 要素を 2 つずつ取り出して関係を評価する手法。ISM と AHP の基礎 |
| **バイナリ行列** | 要素が 0 / 1 のみの行列。ISM の隣接行列がこれ |
| **ブール代数演算則** | 0 / 1 の論理演算（AND, OR）。可到達行列の計算で使う |
| **推移律** | iRj かつ jRk なら iRk が成り立つ性質。ISM の前提 |
| **収束** | M = A+I の冪を繰り返すと、ある回数で M^k = M^(k+1) になる現象 |
| **可到達行列** | i から j へ何らかの経路で到達可能かを示す行列。M^(n−1) で得られる |
| **骨格行列（スケルトン行列）** | 冗長な推移関係を除いた最小構造関係 |
| **隣接行列** | a_ij = 1 で i → j に直接エッジがある行列 |
| **直接影響行列** | DEMATEL の入力。i から j への影響を 0〜4 で評価 |
| **間接的影響行列** | DEMATEL で N², N³, ... の冪和で得られる、間接経由の影響 |
| **影響度 D−R** | 発信力 D（行和）− 被影響力 R（列和）。正なら原因系、負なら結果系 |
| **関連度 D+R** | 発信力 + 被影響力。システム内での中心性 |
| **DSM**（Design Structure Matrix） | 設計要素間の依存関係を行列化する手法。製品設計で使う |
| **MDM**（Multiple Domain Matrix） | 複数ドメイン間の関係を扱う行列。組織×製品×プロセス等 |

### 関連性

```
構造化手法
  ├─ ISM: バイナリ行列 + 一対比較 + 推移律 + 可到達行列 + 骨格行列
  ├─ DEMATEL: 直接影響行列 + 間接的影響行列 + 影響度/関連度
  ├─ DSM: 設計要素の依存関係
  └─ MDM: 複数ドメインの統合

ISM のプロセス
  一対比較 → 隣接行列（バイナリ行列）→ M = A+I のブール代数演算 → 収束 → 可到達行列 → 骨格行列（階層図）

DEMATEL のプロセス
  一対比較（0〜4）→ 直接影響行列 X → 正規化 N → 間接的影響行列 T = N(I−N)⁻¹ → 影響度 D−R + 関連度 D+R

ISM vs DEMATEL
  ISM: 0/1, 推移律あり, 階層化が目的
  DEMATEL: 0〜4, 推移律なし, 影響度の定量が目的
```

### 論述例（10 用語版・約 250 字 × 配点 20〜30 点）

> □構造化手法□ とは、<u>要素間の関係を行列で表現し、複雑なシステムの構造を可視化する手法群</u>である。代表的なのが ISM 法と DEMATEL 法で、=どちらも □一対比較□ を入力とする= が、=扱う関係の種類が異なる=。
>
> ISM 法では、要素間の関係を 0/1 の □バイナリ行列□ で表し、これを □隣接行列□ として扱う。=□推移律□ を仮定する= ことで、=□ブール代数演算則□ に従って M = A + I の冪を計算= し、ある回数で □収束□ する。これにより □可到達行列□ が得られ、=冗長な推移関係を除いた □骨格行列□ で階層構造を可視化する=。
>
> 一方 DEMATEL 法では、=影響の強さを 0〜4 で評価した □直接影響行列□ を入力= とし、=N の冪和で □間接的影響行列□ も含めた総合影響を計算= する。発信力 D と被影響力 R から、=□影響度（D−R）□ で原因系/結果系を、□関連度（D+R）□ でシステム内の中心性= を判別する。

## カテゴリ 3：因果ループ・SD（第 3 回）

過去問頻出用語：**因果ループ図・システムダイナミクス・ストック・フロー・コンバータ・強化ループ R・バランスループ B・S/O リンク**

### 用語の意味

| 用語 | 説明 |
|------|------|
| **因果ループ図（CLD）** | 要素間の因果と符号を可視化した図。S/O リンクで表す |
| **S リンク** | 同方向変化（A 増 → B 増） |
| **O リンク** | 逆方向変化（A 増 → B 減） |
| **強化ループ R** | 自己強化・発散するループ。O が偶数個 |
| **バランスループ B** | 均衡・収束するループ。O が奇数個 |
| **システムダイナミクス（SD）** | 因果ループを定量化し、時間発展をシミュレーションする手法 |
| **ストック** | 時点で測れる蓄積量（状態変数）。例：人口・在庫 |
| **フロー** | 単位時間あたりの増減量。例：出生率・出荷率 |
| **コンバータ** | SD の補助変数。フローやストックを計算する中間値 |

### 関連性

```
因果ループ図
  ├─ S リンク（同方向）・O リンク（逆方向）で符号付き
  ├─ 強化ループ R（O 偶数個）
  ├─ バランスループ B（O 奇数個）
  └─ 質的な分析。定量化には SD が必要

システムダイナミクス
  ├─ 因果ループ図を定量化
  ├─ ストック（蓄積量、状態変数）
  ├─ フロー（変化率、ストックを変える）
  └─ コンバータ（補助変数）

ストック・フロー関係
  ストックの時間微分 = 流入フロー − 流出フロー
  ストックは過去の蓄積を持つので慣性・遅延を生む

因果ループ図 vs 構造化手法
  因果ループ図: 時間変化・フィードバックの可視化（動的）
  構造化手法（ISM/DEMATEL）: 関係の階層・強度の整理（静的）
```

### 論述例（因果ループ + SD・約 200 字 × 配点 15〜20 点）

> □因果ループ図□ は <u>システム内の要素間の因果関係と符号を可視化した図</u>で、=同方向変化を表す □S リンク□、逆方向変化を表す □O リンク□ で結ぶ=。<u>=O が偶数個のループは自己強化する □強化ループ R□、奇数個は均衡する □バランスループ B□ となる=</u>。
>
> 因果ループ図を定量化したのが □システムダイナミクス□ で、<u>=□ストック□（蓄積量、状態変数）と □フロー□（単位時間あたりの増減）の関係でモデル化する=</u>。<u>=ストックは過去の蓄積を持つため、システムに慣性・遅延を生む=</u>。補助変数として □コンバータ□ も使われる。これにより =政策実験や長期予測= が可能になる。

## カテゴリ 4：ファジィ理論（第 4-5 回）

過去問頻出用語：**ファジィ概念・ファジィ集合・ファジィ数・ファジィ関係・メンバーシップ関数**

### 用語の意味

| 用語 | 説明 |
|------|------|
| **ファジィ概念** | 境界の曖昧な概念（「高い」「重い」「若い」など） |
| **ファジィ集合** | 所属の度合いを [0, 1] で表す集合 |
| **ファジィ数** | ファジィ集合で表される数（「だいたい 5」「約 10」など） |
| **ファジィ関係** | 要素間の関係をファジィ集合で表したもの |
| **メンバーシップ関数 μ_A(x)** | x が集合 A に属する度合いを [0, 1] で表す関数 |

### 関連性

```
ファジィ概念
  └─ 数学的に扱うために ファジィ集合 を導入
        └─ メンバーシップ関数 μ_A(x) ∈ [0, 1] で度合いを表現
              ├─ μ = 1：完全に所属
              ├─ μ = 0：完全に非所属
              └─ 中間値：部分的所属

ファジィ集合の特殊形
  ├─ ファジィ数：「だいたい 5」「約 10」などの数
  └─ ファジィ関係：要素間の関係をファジィに表現

ファジィ vs 確率
  確率：事象の発生の不確実性（雨が降る確率 60%）
  ファジィ：概念の境界の曖昧さ（「高齢」の度合い 0.7）
```

### 論述例（ファジィ全用語・約 150 字 × 配点 15 点）

> □ファジィ概念□ とは、<u>「高い」「重い」「若い」のように境界が曖昧な概念</u>を指す。これを数学的に扱うために □ファジィ集合□ が導入され、=要素の所属度を 0 から 1 までの実数で表す □メンバーシップ関数□ μ_A(x) を用いる=。<u>=μ = 1 は完全に所属、μ = 0 は完全に非所属、中間値は部分的所属を意味する=</u>。
>
> □ファジィ集合□ の特殊形として、<u>「だいたい 5」「約 10」のような数を表す □ファジィ数□、要素間の関係を表す □ファジィ関係□ がある</u>。=これらは確率とは異なり、発生の不確実性ではなく概念の境界の曖昧さを扱う=。

## カテゴリ 5：スマートシティ題材の応用論述（2025 年型）

2025 年は「スマートシティの電力需給管理システム」を題材に、構造化手法 / DEMATEL / 因果ループ / SD / ファジィ をどう活用するかが問われた。**汎用テンプレ**として準備。

### 設定の典型パターン

複雑な要求（電力供給の変動、再生可能エネルギー、停電時の最少化、設備投資コスト、需要予測の不確実性、設計上の評価軸：CO₂ 削減、コスト最小化、安定供給）を扱うシステム。

### 各手法の活用例

#### ISM 法の活用

> □ISM 法□ を活用すると、<u>=電力需給管理の要素（需要、供給、蓄電、価格、CO₂ 排出、政策、ユーザー行動など）の依存関係を一対比較で評価し、可到達行列から階層化できる=</u>。例えば、=「政策」が「価格」を、「価格」が「需要」を、「需要」が「供給」を決めるという階層= が浮かび上がる。=これにより、どの要素に介入すれば全体に影響するかが分かる=。

#### DEMATEL 法の活用

> □DEMATEL 法□ を活用すると、<u>=各要素の影響の強さを 0〜4 で評価し、直接影響と間接影響を合わせた総合影響を計算できる=</u>。=影響度 D−R で原因系（介入すべき根本要素）と結果系（症状）を分離= でき、=関連度 D+R でシステムの中心要素を特定= できる。例えば、=「再生可能エネルギーの導入率」が D−R 大なら最優先で扱うべき要素= と判断できる。

#### 因果ループ図の活用

> □因果ループ図□ を活用すると、<u>=要素間のフィードバック構造を S/O 符号で表せる=</u>。例えば、=「需要↑ → 価格↑ → 需要↓」というバランスループ、「再エネ導入↑ → CO₂↓ → 政策支援↑ → 再エネ導入↑」という強化ループ= を可視化できる。これにより、=どこに対症療法が効き、どこに根本介入が必要かが判別= できる。

#### システムダイナミクスの活用

> □システムダイナミクス□ を活用すると、<u>=因果ループ図をストック・フロー・コンバータで定量化し、時間発展をシミュレーションできる=</u>。例えば、=蓄電量（ストック）に対し、発電フローと消費フロー= をモデル化し、=10 年後の需給バランスや CO₂ 削減効果を予測= できる。=政策介入の効果を試算する政策実験= が可能になる。

#### ファジィ理論の活用

> □ファジィ理論□ を活用すると、<u>=「需要が多い」「電力が逼迫している」「停電リスクが高い」など、境界が曖昧な状態を数学的に扱える=</u>。=メンバーシップ関数 μ で 0〜1 の度合いを表現= し、=ファジィ推論で「需要が高くて供給が低いなら蓄電池を放電」などのルール= を実装できる。これにより、=不確実な状況下でも合理的な制御判断= ができる。

## 用語論述の鉄則（試験中）

1. **問題文の用語をすべて使う**：1 つでも漏らさないこと。用語 1 つで 1 点
2. **関連性を 2 重線で示す**：これが 2 点
3. **内容を下線で示す**：これが 1 点（採点者が見つけやすくなる）
4. **問題文に書かれた配点ルール**を必ず読む（例：「1 点 / 内容、2 点 / 関連性、最高 50 点」）
5. **時間がなくても、用語と関連性を箇条書きでも書く**：白紙より遥かにマシ

## 持ち込み時の使い方

1. 問題文の用語を素早く確認
2. このファイルで該当カテゴリを開く
3. 「関連性」のセクションを写経しつつ、問題文の用語に置き換える
4. 「論述例」を骨格にして肉付けする
