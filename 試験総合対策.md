# システム工学基礎 — 試験総合対策（用語の意味と流れ）

この科目で出てくる用語の**意味**と、それらが**どういう問題意識から生まれて、どこでどう使われるか**を、流れに沿って説明するファイル。

「言葉の定義」だけだと結局何を言っているか分からなくなるので、

> **なぜそれを考える必要があるのか → どんな概念か → 具体的にどう使うか → 関連する概念とどう繋がるか**

の順で書く。試験で「説明せよ」と問われたとき、この流れで書ければそのまま答案になる。

## 全体の見取り図（この科目で何をやっているのか）

この科目のテーマは一行で言うと、

> **複雑で曖昧な現実の問題を、どう「システム」として捉えてモデル化するか**

である。

現実の問題（環境、人口、組織、まちづくり、感染症など）には、

- **要素が多い**：登場する人や物が多すぎる
- **絡み合っている**：A が B に影響し、B が C に影響し、C がまた A に戻る
- **時間がかかる**：効果が出るまで何年もかかる
- **不確実**：結果がランダムで予測できない

という特徴がある。これを**頭の中だけ**で考えても整理しきれないので、

1. 関係を**図や行列**で書き出して可視化する（**構造分析**）
2. 時間発展を**数学的に**追いかける（**動的分析**）
3. ランダムなものを**確率**で扱う
4. 個体の行動から全体を**シミュレーション**する

という4つのアプローチが必要になる。本科目はこれらの道具立てを順番に学ぶ。

科目は2部構成：

- **前半（青山担当、第1〜5回）**：構造をどう捉えるか。**ISM, DEMATEL, Cognitive Map, AHP, SD, ファジィ**
- **後半（藤井担当、第6〜12回）**：時間・確率・離散をどう扱うか。**マルコフ、待ち行列、オートマトン、エージェント、SOC、離散化**

各回の用語は、この全体の流れのどこにある話かを意識すると意味が分かりやすい。

## 第1回：ガイダンス — 「システム」とは何か

### なぜここから始まるのか

「システム」という言葉は日常で多用されるが、工学では厳密な定義が必要。**何をシステムとして扱うか**を決めないと、分析が始まらない。

### システムの3要素

**システム** = 次の3つを持つ集合体：

1. **要素**：システムを構成する個々の実体（人、物、情報など）
2. **相互作用**：要素間の関係・因果・物質や情報の流れ
3. **目的性**：システム全体が達成しようとする機能や目標

例：自動車システムなら、要素はエンジン・タイヤ・運転手など、相互作用は燃料の流れや力の伝達、目的性は「人を運ぶ」。

「要素だけ」ではシステムではない（バラバラの部品の山）。「相互作用がある」「目的を持つ」ことで初めてシステムと呼べる。

### 境界の引き方：存在論 vs 認識論

システムを扱うときに必ず問題になるのが**境界**。「ここからここまでがこのシステム」と決める必要がある。この境界の引き方には2つの立場がある。

**存在論的境界**：客観世界に**実在する区切り**。細胞膜、国境、企業の法人格など、物理的・制度的にはっきり区切られているもの。

**認識論的境界**：分析者が**モデル化のために設定する区切り**。同じ対象でも、観点や問題意識で境界が変わる。

例：「学生」を分析するとき、

- 大学に在籍している人全員、と境界を引く（存在論的）
- 「経済的に独立した人を除く」「30歳未満に限る」など、目的に応じて切る（認識論的）

→ 重要なのは「**境界の設定は分析の出発点**」であり、これを明示することが分析の透明性を保つカギ。「私はこういう境界で考える」と最初に宣言する。

### 良定義問題と悪定義問題

問題には2種類ある。

**良定義問題**：目標・手段・評価基準が明確に決まっている問題。例：最短経路を求める、線形計画法。**最適解が存在**し、計算で求められる。

**悪定義問題（wicked problem）**：目的・関係者・解が曖昧で、合意形成が困難な問題。例：気候変動対策、貧困問題、まちづくり、教育改革。

悪定義問題の特徴：

- 何が「正しい解」か関係者で意見が分かれる
- 解は「**正しい/間違い**」ではなく「**より良い/悪い**」で評価される
- 問題自体が**やってみないと分からない**

→ こういう問題には数学的最適化は使えない。代わりに **ソフトシステムズ方法論（Checkland）** のような、**関係者の世界観を引き出しながら学習していくプロセス**が必要。**構造化手法**（ISM、DEMATEL など）はこの場面で活躍する。

### 構造モデリング（SM）と動的モデリング（DM）の使い分け

システムを扱う2つのアプローチ。

**構造モデリング（Structural Modeling）**：要素間の**関係・階層**を整理することに焦点。時間軸は持たない（**静的**）。

- 代表的手法：ISM、DEMATEL、Cognitive Map、AHP
- 問い：「**何が何に影響するか**」「**どんな構造になっているか**」
- 用途：関係者の合意形成、本質的な構造の把握

**動的モデリング（Dynamic Modeling）**：要素の状態が**時間とともにどう変わるか**に焦点。**時系列・フィードバック**を扱う。

- 代表的手法：システムダイナミクス（SD）、エージェントシミュレーション
- 問い：「**時間とともにどう変化するか**」「**この政策を導入したら10年後どうなるか**」
- 用途：政策実験、シナリオ分析、長期予測

両者は**補完関係**。まず構造を整理（SM）してから、時間発展をシミュレーション（DM）する流れが典型。

### システムズアプローチのプロセス

問題解決の標準的な流れ：

1. **問題把握**：何が問題か、誰が困っているかを明らかにする
2. **解決策の構想・評価**：候補を出し、技術・経済・社会の3面から評価
3. **試行と評価**：実際にやってみて結果を見る
4. **改善の反復**：結果を元に解決策を修正

これは1回で終わらず、**繰り返し**ながら学習していく。

## 第2回：構造を行列で扱う — グラフ・ISM・DEMATEL

### なぜ構造を可視化するのか

例えば「投票率を上げたい」という問題で、関係する要素を考える：

- 政治への関心、候補者情報の入手しやすさ、投票所のアクセス、SNS の影響、家族の習慣、過去の投票経験……

10 個ぐらいすぐ出る。これらが**互いにどう影響しあっているか**を頭の中だけで整理するのは無理。

→ **行列**（数字の表）にして書き出せば、コンピュータで計算できる。これが構造分析の出発点。

### 隣接行列：関係を 0 と 1 で表す

要素を1番、2番、3番…と番号付けて、

> **「要素 i から要素 j に影響がある」なら a_{ij} = 1、なければ 0**

としたものを**隣接行列 A** と呼ぶ。これで「誰が誰に影響するか」が表になる。

### 行列の冪と経路

隣接行列を2乗、3乗すると面白いことが起きる。

**A の k 乗の (i, j) 成分** = 「i から j への**長さ k の経路の本数**」

つまり A² なら「2 ステップで行ける経路の数」、A³ なら「3 ステップ」を表す。

これを使うと「**i から何ステップでもいいから j に到達できるか**」が分かる。これが**可到達**の概念。

### 可到達行列

「自分自身も到達可能」とするために、隣接行列に単位行列 I を足す：

> **M = A + I**

すると M^k は「**i から j への長さ 0〜k の経路の数**」を意味する。

要素が n 個あるとき、最長の単純経路（同じ点を通らない経路）は n−1 ステップなので、**M^(n−1)** で可到達関係がすべて捉えられる。これが**可到達行列**。

実務的には M, M², M³ … と順に計算して、**変化が止まった時点で収束**したと判断する。

### ISM（Interpretive Structural Modeling）：階層化の手法

可到達行列が手に入ったら、次にやるのは**階層化**。「**どの要素が上位（根本原因）で、どの要素が下位（結果）か**」を整理する。

#### 推移律という前提

ISM では「**推移律**」を前提とする：

> **i が j に影響し、j が k に影響するなら、i は k にも影響する**

例：A が B の上司、B が C の上司なら、A は C の上司（推移的）。

ただし**すべての関係が推移的とは限らない**。

- 友人関係：A と B が友人、B と C が友人でも、A と C は友人とは限らない
- じゃんけん：グー > チョキ、チョキ > パー、でもグーはパーに負ける

→ ISM を使う前に「**この関係は推移的か**」を確認する必要がある。推移的でないなら DEMATEL や Cognitive Map を使う。

#### レベル抽出のロジック

ISM では各要素 i について2つの集合を考える：

- **R(i)：到達可能集合** = i から到達できる要素の集合（i 自身も含む）。「**i の下流**」
- **A(i)：先行集合** = i に到達できる要素の集合（i 自身も含む）。「**i の上流**」

階層を抽出する条件：

> **R(i) ∩ A(i) = R(i)** となる要素 i は、**最上位レベル**にある

言葉で言うと「**i から到達できる要素はすべて i にも到達できる**」つまり「**i から下流に出られない＝もう下に要素がない＝最上位**」。

このアルゴリズム：

1. 全要素について R∩A=R を判定 → 該当する要素を1レベルとして抽出
2. 抽出した要素を除いて、残りの要素で同じ判定を繰り返す
3. すべての要素が階層化されるまで反復

これで「**上位ブロック → 下位ブロック**」の階層図ができる。

#### ブロック三角化と骨格行列

**ブロック三角化**：強連結成分（互いに到達可能なノード群）をまとめ、階層順に並べ替える。これで全体構造が見やすくなる。

**骨格行列**：A→B→C のとき、推移律で派生する A→C のような**冗長な関係**を除いた**最小構造関係**。可読性を上げるために描画では骨格行列を使う。

### DEMATEL：影響の強さを定量化する

ISM は「**ある/なし**」の0/1関係しか扱えない。実際は「強い影響」「弱い影響」と程度がある。これを扱うのが **DEMATEL（Decision Making Trial and Evaluation Laboratory）**。Battelle Geneva 研究所が 1972 年に開発した。

#### 直接影響行列 X

各要素から各要素への影響を **0〜4** で評価する：

- 0：影響なし
- 1：弱い影響
- 2：中程度の影響
- 3：強い影響
- 4：非常に強い影響

これで **直接影響行列 X**（n×n の正方行列）ができる。

#### 正規化 N

直接影響行列を**正規化**する：

$$
N = \frac{X}{\max(\text{各行の行和})}
$$

つまり「各行の行和」を計算して、その中で最大のもので全体を割る。

なぜ正規化するか：次のステップで N の冪和を取るが、N が小さくないと級数が発散する。**N の各要素を 0〜1 に収める**ためにスケーリングが必要。

#### 総合影響行列 T

ここが DEMATEL の核心。直接影響だけでなく、**1段階間接、2段階間接、…の影響もすべて足し合わせる**：

$$
T = N + N^2 + N^3 + \cdots = N(I - N)^{-1}
$$

意味：

- N：1段階の直接影響
- N²：2段階の間接影響（i → 中継 → j）
- N³：3段階の間接影響
- …

を全部合計したのが T。これにより「**i が直接 j に影響していなくても、別の要素を介して間接的に影響している場合**」も捕捉できる。

#### D と R：発信力と被影響力

T から2つの指標を取り出す：

- **D = T の各行の和**：要素 i が他のすべての要素に与える総影響量（**発信力**）
- **R = T の各列の和**：要素 i が他のすべての要素から受ける総影響量（**被影響力**）

#### D + R と D − R：関連度と因果性

- **D + R（関連度）**：その要素がシステム内でどれだけ**関わりが多い**か。大きいほど中心的な要素
- **D − R（因果性）**：その要素が**原因系か結果系か**を判別
  - **D − R > 0**：発信が多い → **原因系**（介入すべき根本原因）
  - **D − R < 0**：受信が多い → **結果系**（対症療法の対象になる症状）

#### DEMATEL の実用的な使い方

- ① で原因系の要素（D−R が大きい）を特定 → **そこに介入する**
- ② で結果系の要素（D−R が小さい）は症状 → 介入しても根本解決にならない
- ③ 関連度（D+R）が大きい要素は**システムの中心**→ 注目すべき

#### ISM と DEMATEL の使い分け

| | ISM | DEMATEL |
|---|-----|---------|
| 関係の表現 | 0/1（あり/なし） | 0〜4（強さ） |
| 推移律 | **仮定する** | 仮定しない |
| 主目的 | 階層化 | 影響度・原因/結果の定量 |
| 出力 | 階層図 | D−R/D+R グラフ |

ISM で全体構造を捉えてから DEMATEL で重要要素を絞り込む、という併用もよく行われる。

## 第3回：因果ループと SD — 時間とフィードバックを扱う

### なぜ時間が必要か

ISM や DEMATEL は**静的**（時間軸を持たない）。でも現実の問題には**時間**が絡む：

- 政策を導入してから10年後、人口はどうなる？
- ダイエットを始めて3ヶ月後、体重はどう変化する？
- 感染症対策を打ったら1年後どうなる？

こういう**動的な挙動**を扱うのが**システムダイナミクス（SD）**。MIT の Jay Forrester が 1961 年に開発した。

### 因果ループ図（CLD）：質的に時間を捉える

SD の質的記法。要素間に矢印を引き、**符号**を付ける：

- **S（Same）**：同じ方向に動く。A が増えれば B も増える、A が減れば B も減る
- **O（Opposite）**：逆方向に動く。A が増えれば B は減る、A が減れば B は増える

例：

- 「勉強時間 →(S) 学力」（勉強が増えれば学力が増える）
- 「価格 →(O) 需要」（価格が上がれば需要が減る）

### 強化ループ（R）とバランスループ（B）

矢印を辿って一周してくる**ループ**ができることがある。これがシステム思考の核心。ループには2種類ある。

**強化ループ（R: Reinforcing）**：自己強化する。一度動き出すと**発散・暴走**していく。

例：「貯金 → 利息 → 貯金がさらに増える → さらに利息……」（複利のループ）

**バランスループ（B: Balancing）**：均衡に**収束**する。安定化させる効果。

例：「体温が上がる → 発汗 → 体温が下がる」（体温調節のループ）

#### R/B の判定方法

ループ内の **O の符号**を数える：

- **O が偶数個** → R（強化ループ）
- **O が奇数個** → B（バランスループ）

数学的理由：S を +1、O を −1 として、ループ全体の積を計算する。

- 積が +1 なら、一周回って同じ方向に戻る → **自己強化（R）**
- 積が −1 なら、一周回って逆方向に戻る → **打ち消し合う（B）**

例：「ストレス↑ →(O) 睡眠↓ →(O) 疲労↑ →(S) ストレス↑」

O が2個（偶数）→ **R**。実際にトレースしてみると、ストレス↑ → 睡眠↓ → 疲労↑ → さらにストレス↑ となり、自己強化の悪循環。

### ストックとフロー：定量的な SD

因果ループ図は質的だが、**定量シミュレーション**するには **ストック・フロー** という概念が要る。

**ストック（stock）**：時点で測れる**蓄積量**。状態変数。

- 例：人口、湖の水量、CO₂濃度、口座残高、在庫量

**フロー（flow）**：単位時間あたりの**増減量**。

- 例：出生率、流入量、CO₂排出率、入金額、出荷量

両者の関係：

> **ストックの時間微分 = 流入フロー − 流出フロー**

つまりストックは「**フローを時間で積分**」したもの。

例：人口 = ∫(出生 − 死亡 + 転入 − 転出) dt

ストックは**過去の蓄積**を持つので、システムに**慣性・遅延**を生む。「税制を変えてもすぐ人口は変わらない」のは、人口というストックが過去の蓄積だから。

### システム原型（Senge）

Peter Senge は「The Fifth Discipline（第5の規律）」で、SD でよく出てくる典型パターンを **システム原型** として整理した。代表的なもの：

#### Fix That Fails（応急処置の失敗）

| ループ | 役割 |
|--------|------|
| B（即効） | 症状を一時的に軽減する |
| R（副作用） | 根本原因が悪化する |

例：

- **借金で生活費を補填**：当面楽（B）→ 利息で長期困窮（R）
- **抗生物質乱用**：感染症は治る（B）→ 耐性菌が増える（R）
- **道路拡張で渋滞緩和**：一時的に流れる（B）→ 誘発交通で混雑悪化（R）

**示唆**：応急処置は**短期的解決と長期的悪化のトレードオフ**を生む。根本原因への介入が必要。

#### その他の代表的な原型

- **Shifting the Burden（負担の転嫁）**：症状対処に頼って根本原因への対処が衰える
- **Limits to Growth（成長の限界）**：成長ループに必ず制約ループが現れて頭打ちになる
- **Tragedy of the Commons（共有地の悲劇）**：個人合理性が共有資源を破壊する
- **Eroding Goals（目標の侵食）**：目標が現実に合わせて下がっていく

### SD の代表例

- **World3 モデル**（Limits to Growth, 1972）：人口・資源・汚染・食糧・工業の世界モデル。「成長の限界」を予測した古典
- **Urban Dynamics**（Forrester 1969）：都市の発展・衰退をモデル化

## 第4回：ファジィ理論 — 「曖昧さ」を数学で扱う

### なぜファジィが必要か

「高齢」「速い」「美味しい」「暑い」。これらは日常で普通に使う言葉だが、**境界がはっきりしない**。

- 何歳から「高齢」？
- 時速何キロから「速い」？

確率とは違う性質。「**雨が降る確率 60%**」は雨が降るか降らないかの**発生**の不確実性。「**この人は『高齢』に 70% 属する**」は概念自体の**境界の曖昧さ**。

これを数学で扱うのが**ファジィ理論**（Zadeh、1965）。

### メンバーシップ関数

普通の集合（クラシック集合）は「**属する/属さない**」の二択（0 か 1）。

ファジィ集合では、要素が集合に属する**程度**を 0 から 1 までの実数で表す。これを **メンバーシップ関数 μ_A(x)** と呼ぶ。

- μ = 1：完全に属する
- μ = 0：完全に属さない
- 中間値：部分的に属する

例：「若い」という集合のメンバーシップ：

- μ(20歳) = 1.0
- μ(25歳) = 0.9
- μ(40歳) = 0.5
- μ(60歳) = 0.1

### 確率とファジィの違い

| | 確率 | ファジィ |
|---|------|----------|
| 対象 | 事象の**発生**の不確実性 | 概念の**曖昧さ・程度** |
| 例 | 雨が降る確率 60% | 「高齢」の度合い 0.7 |
| 解釈 | 何度も繰り返したら 60% で起きる | 「高齢」概念に 70% 属する |

両者は**別の不確実性**を扱う。

### ファジィ演算と推論

ファジィ集合に対する論理演算：

- **AND**：μ_A AND μ_B = min(μ_A, μ_B)
- **OR**：μ_A OR μ_B = max(μ_A, μ_B)
- **NOT**：NOT μ_A = 1 − μ_A

これを使った**ファジィ推論**の流れ：

1. **ファジィ化**：明確な入力値（例：温度 28℃）→ メンバーシップ度（「やや暑い」0.7）
2. **ルール適用**：「もし暑いなら冷房を強くせよ」のようなルール群を適用
3. **合成**：複数ルールの結果を集約
4. **デファジィ化**：ファジィな結果 → 明確な出力値（重心法など）

応用例：エアコンの自動制御、洗濯機、新幹線の自動運転制御。

### FCM（Fuzzy Cognitive Map）

Kosko（1986）が提案。Cognitive Map（符号付きの因果グラフ）に**ファジィな重み**（−1 から +1 の値）を持たせたもの。

- Cognitive Map：因果の**符号**のみ
- DEMATEL：因果の**強さ**のみ（非負）
- FCM：**符号と強さ**の両方

ファジィ行列演算 + 反復計算で系の挙動を予測する。

## 第5回：AHP — 複数の選択肢から1つを選ぶ

### なぜ AHP が必要か

「3つの提案のうちどれを採用するか」「複数の評価基準で総合的に判断したい」という場面はよくある。例えば：

- 新車を買うとき：価格・燃費・安全性・デザインで総合判断
- 公共事業の候補地選定：コスト・環境影響・利便性・住民賛成
- 部署の人事評価：技術力・コミュニケーション・成果

複数の基準を持って複数の選択肢から1つを選ぶ問題を **多基準意思決定** と呼び、その代表手法が **AHP（Analytic Hierarchy Process）**。Saaty が 1980 年に開発した。

### 階層構造

AHP では問題を3層の階層に整理する：

1. **目標**（最上位）：例「最適な新車を選ぶ」
2. **評価基準**（中間）：価格、燃費、安全性、デザイン
3. **代替案**（最下位）：車 A、車 B、車 C

### 一対比較

評価基準同士、代替案同士を**2つずつ取り出して**重要度を比較する。Saaty の **1〜9 尺度**：

| 値 | 意味 |
|----|------|
| 1 | 同等 |
| 3 | やや優 |
| 5 | 優 |
| 7 | かなり優 |
| 9 | 極めて優 |

例：「価格 vs 燃費」で「価格がやや重要」と判断 → 3。

これを n×n の**比較行列**にまとめる。「価格 vs 燃費」が 3 なら、対称位置「燃費 vs 価格」は 1/3（逆数）。

### 重みの算出

比較行列の**最大固有値**に対応する**固有ベクトル**を計算し、正規化して**重みベクトル**を得る。これが各基準の相対的重要度を表す。

簡略法：各行の幾何平均を取って正規化する方法もある。

### 整合性指標 CR

人間の判断は完全に整合的ではない。「価格 > 燃費」「燃費 > 安全性」「安全性 > 価格」のように**循環矛盾**を起こすことがある。これを検出するのが **CR（Consistency Ratio）**：

> **CR < 0.1** なら判断は十分一貫している

CR が 0.1 を超えたら、判断を見直す必要がある。

### AHP の注意点

- 一対比較数は n(n−1)/2。要素 10 個なら 45 回の比較が必要 → **負担大**
- **ランキング逆転問題**：代替案を1つ追加すると、既存の順位が変わることがある（一見おかしい）

### 他手法との使い分け

| 手法 | 問い | 出力 |
|------|------|------|
| ISM | 要素間の構造・階層は？ | 階層図 |
| DEMATEL | 影響の強さ・原因/結果は？ | D−R グラフ |
| Cognitive Map | 因果の正負は？ | 符号付きグラフ |
| **AHP** | **どの案を選ぶか？** | **優先順位** |

## 第6回：確率モデルの基礎

### なぜ確率が必要か

ここから後半（藤井担当）。前半は「静的な構造」を扱った。後半は「**不確実性と時間発展**」を扱う。

現実の多くの現象は**ランダム**：

- 顧客がいつ来店するか
- 機械がいつ故障するか
- 株価が明日どうなるか
- 感染症が広がるかどうか

こういう現象を**確率モデル**で記述し、**シミュレーション**で挙動を予測するのが後半の主題。

### 確率の基礎用語

- **試行**：結果が偶然に決まる実験（コイン投げ、サイコロ振り）
- **標本空間 Ω**：すべての結果の集合（コイン投げなら {表, 裏}）
- **事象**：標本空間の部分集合（「表が出る」など）
- **確率 P(A)**：事象 A が起きる度合い、0 ≤ P(A) ≤ 1

#### Kolmogorov の確率公理

確率の数学的基礎を Kolmogorov が以下の3つの公理にまとめた：

1. 任意の事象 A について **0 ≤ P(A) ≤ 1**
2. **P(Ω) = 1**（必ず何かが起きる）
3. **可算加法性**：互いに排反な事象 A_1, A_2, ... について P(∪A_i) = Σ P(A_i)

これだけから確率論のすべてが導かれる。

#### 条件付き確率とベイズの定理

「B が起きたとき A が起きる確率」を**条件付き確率**と呼ぶ：

$$
P(A \mid B) = \frac{P(A \cap B)}{P(B)}
$$

これから導かれるのが**ベイズの定理**：

$$
P(A \mid B) = \frac{P(B \mid A) P(A)}{P(B)}
$$

応用：医療診断（症状から病気の確率を推定）、スパムフィルタ、機械学習の Naive Bayes 分類器。

### 確率モデルと確率過程

**確率モデル**：確率的なルールで事象を記述したモデル全般。

**確率過程**：状態が**時間とともに変化する**確率モデル。{X_t}_{t∈T} という時間ごとの確率変数の族で表す。

確率過程は確率モデルの一種で、時間軸を持つもの。次のマルコフ連鎖、待ち行列はすべて確率過程。

### マルコフ性

確率過程の中で**特別扱いが楽になる**性質が**マルコフ性**：

> **次の状態は、現在の状態だけで決まる。過去の履歴は不要**

数式で書くと：

$$
P(X_{n+1} = j \mid X_n = i, X_{n-1}, \ldots, X_0) = P(X_{n+1} = j \mid X_n = i)
$$

つまり「**現在さえ分かれば過去は忘れていい**」。

なぜこれが嬉しいか：過去の全履歴を覚えなくても、遷移確率だけで未来を予測できる。**計算がシンプルになる**。

例：

- **ランダムウォーク**：現在の位置だけ分かれば、次の位置の確率が決まる
- **天気の遷移**：今日が晴れなら、明日の晴れ確率は決まる
- **ギャンブラーの破産問題**：現在の所持金だけで次の勝敗が決まる

多くの現象がマルコフ近似で十分に近似できるのが、マルコフモデルの強み。

## 第7回：確率分布

### 分布関数：累積で見る

確率変数 X について、

$$
F(x) = P(X \le x)
$$

を **分布関数（CDF, Cumulative Distribution Function）** と呼ぶ。「X が x 以下である確率」。

性質：

1. **単調非減少**：x_1 ≤ x_2 なら F(x_1) ≤ F(x_2)
2. **右連続**：F(x+) = F(x)
3. **F(−∞) = 0, F(∞) = 1**

これは離散型・連続型のどちらの確率変数にも定義できる。

### 離散型と連続型

**離散型**：X が「0, 1, 2, ...」のような飛び飛びの値しか取らない。例：サイコロの目、客の人数。

- 確率質量関数 p(x) = P(X = x)
- 期待値 E[X] = Σ x_i p(x_i)

**連続型**：X が実数のあらゆる値を取りうる。例：身長、待ち時間。

- 確率密度関数 f(x) ≥ 0、∫ f(x) dx = 1
- 期待値 E[X] = ∫ x f(x) dx
- **注意**：連続型では「ピッタリ x である確率」は 0。確率は**区間の積分**で与える

### 代表的な分布

| 分布 | 用途 | 期待値 | 分散 |
|------|------|--------|------|
| **ベルヌーイ** | 1回の成功/失敗 | p | p(1−p) |
| **二項** B(n,p) | n 回独立試行の成功数 | np | np(1−p) |
| **ポアソン** | 単位時間に稀な事象が起きる回数 | λ | λ |
| **幾何** | 初成功までの試行回数 | 1/p | (1−p)/p² |
| **指数** | 事象間隔（待ち時間） | 1/λ | 1/λ² |
| **正規** | 測定誤差、自然現象 | μ | σ² |

#### 指数分布の無記憶性

指数分布には特別な性質がある：

$$
P(X > s + t \mid X > s) = P(X > t)
$$

意味：「**これまで s 時間待った**」という情報が、残り待ち時間の分布に**影響しない**。

例：バスを 10 分待ったが来ない。さらに 5 分待つ確率は、待ち始めの瞬間に 5 分待つ確率と**同じ**。

これは**マルコフ性と概念的に同じ**。次の状態は現在の状態だけで決まり、過去（これまで何分待ったか）は関係ない。

→ 待ち行列理論（M/M/1）の理論的基盤になる性質。

### モンテカルロ法と中心極限定理

#### モンテカルロ法

複雑な期待値・確率を**乱数サンプル**で近似する方法。理論的に計算できないが、シミュレーションなら可能、という場面で使う。

例：円周率の推定。正方形に乱数で点を打って、円の中に入った割合から π を推定。

サンプル数 N で誤差は **1/√N** のオーダーで小さくなる（大数の法則）。

応用：高次元積分、リスク評価、複雑系シミュレーション。

#### 中心極限定理

独立同分布の確率変数 X_i の和は、n が大きくなると**正規分布に近づく**：

$$
\frac{\bar{X} - \mu}{\sigma / \sqrt{n}} \to N(0, 1)
$$

これは「**個々の分布が何であろうと、たくさん足せば正規分布になる**」という驚くべき法則。統計的推定、モンテカルロ誤差解析の理論的基盤。

## 第8回：マルコフ連鎖と待ち行列

### マルコフ連鎖

#### 推移確率行列

マルコフ連鎖は、状態空間 S = {1, 2, ..., m} を持ち、**マルコフ性**を満たす確率過程。

「状態 i から状態 j に1ステップで遷移する確率」を p_{ij} と書く。これを行列にまとめたのが**推移確率行列 P**：

$$
P = (p_{ij})
$$

- 各 p_{ij} ≥ 0
- **各行の和 = 1**（必ずどこかに遷移する）

#### l ステップ推移

「l ステップ後に状態 i から j にいる確率」は p_{ij}^{(l)} = (P^l)_{ij}。つまり**行列の冪**で l ステップ先の確率が得られる。

これは Chapman-Kolmogorov 方程式から導かれる：

$$
p_{ij}^{(l+n)} = \sum_k p_{ik}^{(l)} p_{kj}^{(n)}
$$

「l ステップで k に行き、その後 n ステップで j に行く確率の総和」。

#### 状態の分類

| 用語 | 意味 |
|------|------|
| **既約**（irreducible） | 全状態が相互に到達可能 |
| **再帰的**（recurrent） | 有限時間で戻ってこれる |
| **過渡的**（transient） | 戻らない確率が正 |
| **非周期的**（aperiodic） | 周期が 1（戻る間隔がバラバラ） |
| **吸収状態** | p_ii = 1。一度入ったら出られない |

#### 定常分布：長期平均の挙動

「長時間後、どの状態にどれくらいの確率でいるか」を表すのが**定常分布 π**：

$$
\pi P = \pi, \quad \sum_i \pi_i = 1
$$

意味：「**1ステップ進んでも分布が変わらない**」状態。**平衡分布**とも呼ぶ。

#### 計算例（試験の定番）

$$
P = \begin{pmatrix} 0.9 & 0.1 \\ 0.2 & 0.8 \end{pmatrix}
$$

πP = π より：

- 0.9 π_1 + 0.2 π_2 = π_1 → 0.1 π_1 = 0.2 π_2 → π_1 = 2 π_2
- π_1 + π_2 = 1 → 2 π_2 + π_2 = 1 → π_2 = 1/3, π_1 = 2/3

よって **π = (2/3, 1/3)**。

→ 長時間後、状態 1 にいる確率が 2/3、状態 2 にいる確率が 1/3 に収束する。**初期状態に依存しない**（既約 + 非周期的のとき）。

### 待ち行列理論

顧客がランダムに到着し、サーバが処理する系を扱う。応用は無限：銀行 ATM、コールセンター、Web サーバー、製造ライン、空港の搭乗ゲート。

Erlang が 1909 年に電話交換機の設計から発展させた。

#### ケンドール記法 A/B/c

- A：到着過程
- B：サービス過程
- c：サーバ数

**M/M/1** は最も基本：

- M（Markov）：到着がポアソン過程（指数分布の到着間隔）
- M：サービス時間も指数分布
- 1：サーバ 1 台

#### 利用率 ρ

$$
\rho = \frac{\lambda}{\mu}
$$

- λ：到着率（1単位時間あたり何人来るか）
- μ：サービス率（1単位時間あたり何人処理できるか）

ρ は「**サーバが忙しい時間の割合**」。

**ρ < 1** なら安定。**ρ ≥ 1** だと到着が処理を上回り、**待ち行列が無限に伸びる**（不安定）。

ρ → 1 に近づくと、待ち時間が急増する。ρ = 0.9 で平均行列長 9、ρ = 0.99 で 99。**実務では稼働率 80% 程度に抑える**のが目安。

#### リトルの法則

待ち行列理論の中で最も一般的な法則：

$$
L = \lambda W
$$

- L：系内の平均顧客数
- λ：到着率
- W：平均滞在時間

意味：「1人が平均 W 時間滞在し、λ 人/時で入ってくるなら、系内の平均人数は λ × W」。

**驚くべきことに、到着過程・サービス過程の分布によらず常に成り立つ**。極めて普遍的。ストックとフローを結ぶ関係。

#### M/M/1 の主要公式

- L = ρ/(1−ρ)：系内平均人数
- W = 1/(μ−λ)：平均滞在時間
- L_q = ρ²/(1−ρ)：平均待ち行列長
- W_q = ρ/(μ−λ)：平均待ち時間

## 第9回：オートマトンとセルオートマトン — 局所ルールから大域へ

### なぜここでオートマトンか

確率モデル（マルコフ・待ち行列）の次に、**離散モデル**を扱う。コンピュータは離散計算しかできないので、シミュレーションは本質的に離散。

オートマトンは「**状態と遷移ルール**を持つ離散系」。

### 有限オートマトン

形式的定義：(Q, Σ, δ, q_0, F)

- Q：状態の集合
- Σ：入力アルファベット
- δ：遷移関数 Q × Σ → Q（「状態 q で入力 a を受けたら、次の状態は何か」）
- q_0：初期状態
- F：受理状態

用途：文法解析、字句解析、コンパイラ、正規表現マッチング。

特徴：**1 つの個体**が、入力に応じて状態を変える。

### セルオートマトン（CA）

ここから話が大きく変わる。**格子状にたくさんのセルが並び、それぞれが同時に局所ルールで更新される**系。

- L：格子（1次元、2次元など）
- S：各セルの状態集合
- N：近傍関係（どのセルが「隣」か）
- f：更新関数

**1 体ではなく多数のセルが、同じルールで一斉に動く**ところがオートマトンとの違い。

#### 近傍の取り方

2次元の格子では2種類：

- **von Neumann 近傍**：上下左右の 4 マス
- **Moore 近傍**：周囲 8 マス（斜めも含む）

#### Game of Life（Conway 1970）

最も有名な2次元 CA。各セルは「生」または「死」の2状態。Moore 近傍の生セル数で更新：

- 生のセルは、隣接が **2 か 3** で生存（それ以外は死亡）
- 死のセルは、隣接が **ちょうど 3** で誕生

このシンプルなルールから、

- **静物**（変化しないパターン）
- **振動子**（周期的に振動）
- **グライダー**（移動するパターン）
- **グライダー銃**（グライダーを永遠に生成する装置）
- 任意の論理回路（チューリング完全）

までが創発する。

#### Wolfram の CA 分類

Stephen Wolfram は1次元 CA を4クラスに分類：

- **Class I**：均一に収束
- **Class II**：周期パターン
- **Class III**：カオス（ランダムに見える）
- **Class IV**：複雑構造（最も興味深い、計算可能性を持つ）

「**単純ルールから複雑挙動が生まれる**」のが CA の本質的な発見。

## 第10回：エージェントベース・モデル

### エージェントとは

**エージェント** = 環境を**知覚**し、自律的に**行動**する個体。

特徴：

- 自分の状態を持つ
- 環境を観察できる
- 目標や効用関数を持つ（場合がある）
- 学習することもある（場合がある）

CA との違い：CA は「同じルールで動く一様なセル」、エージェントは「**個別の判断・目標を持つ自律個体**」。

### エージェントの5タイプ（Russell & Norvig）

AI の標準教科書で整理された分類：

1. **単純反射**：現在の知覚だけで行動（IF-THEN ルール）。例：サーモスタット
2. **モデルベース反射**：内部に世界モデルを持ち、見えない部分を推測
3. **目標ベース**：目標を持ち、目標達成のための手段を選ぶ
4. **効用ベース**：効用関数を持ち、期待効用を最大化する合理的選択
5. **学習エージェント**：経験から自分を改善する

合理性のレベルが上がるにつれて、複雑な状況に対応できる。

### MAS（Multi-Agent System）

複数のエージェントが**相互作用**する系を **MAS（マルチエージェントシステム）** と呼ぶ。

特徴：

- **集中制御がない**：誰も全体を指揮しない
- **局所相互作用**：各エージェントは近くの相手とのみ相互作用
- **創発**：個々の単純ルールから、集団レベルで予測困難な性質が生まれる

### 創発（emergence）

**創発** = 個々の局所ルールから、**集団レベルで予測困難な大域的性質**が生じる現象。

#### 創発の代表例

| 系 | 局所ルール | 大域現象 |
|----|------------|----------|
| **Schelling 分居モデル** | 弱い人種選好 | 強い分居 |
| **Reynolds の boids** | 分離・整列・凝集 | 鳥群の飛行 |
| **アリのフェロモン** | 局所判断 | 最短経路発見 |
| **市場参加者** | 個別売買 | 価格動学・暴落 |
| **渋滞** | 個別追従行動 | 自然渋滞 |

#### Schelling の分居モデル（重要）

Schelling は 1971 年に次のことを示した：

- 各人が「**周囲の少なくとも 30% が同じ人種であってほしい**」という**弱い選好**を持つ
- すると最終的に**強い人種分居**が生まれる

つまり「**個人レベルの差別意識が弱くても、社会レベルでは強い分離が出現する**」。これが創発の典型。意図しない結果。

#### Reynolds の boids

3つの単純ルールだけで鳥の群れ飛行を再現：

1. **分離**：近くの個体から離れる
2. **整列**：周りの個体と方向を揃える
3. **凝集**：群れの中心に近づく

これだけで自然な群れ運動が生まれる。映画 CG にも使われている。

## 第11回：自己組織化臨界（SOC）

### 複雑系の特徴

ここまで学んだ CA、MAS、確率モデルは「**複雑系**」のカテゴリーに入る。複雑系には共通の特徴がある：

- **局所的相互作用から大域的振る舞いが生じる**
- **全体の振る舞いが個々の要素からは明らかでない**
- **べき乗則、フラクタル、1/f ゆらぎが現れる**

これらの背後にある原理が**自己組織化臨界（SOC）**。Bak、Tang、Wiesenfeld が 1987 年に提唱した。

### 臨界状態と SOC

**臨界状態** = 相転移点。氷が水になる温度のような、性質が劇的に変わる境界。

通常の臨界は**外部パラメータ**（温度など）でチューニングして到達する。

**自己組織化臨界（SOC）** = **外部調整なしに、系が自分で臨界状態を保つ**現象。

### サンドパイル・モデル

SOC の典型例。砂を1粒ずつ落とし、4近傍に崩れる単純なルール：

1. ランダムに 1 セルを選び、値を +1（砂が積もる）
2. 値が **4 以上**なら、そのセルから **4 減らし、上下左右の 4 近傍に +1** ずつ加える（崩壊）
3. 崩壊によって他のセルも 4 以上になれば、**連鎖的に崩壊**
4. すべて 3 以下に戻ったら次の砂粒へ

このシンプルなルールから、

- 砂山は自然に**一定の傾き**を保つ（臨界状態）
- 崩壊の規模が**べき乗分布**になる：小さな崩壊から大規模な雪崩まで

### べき乗則

頻度 N(x) が大きさ x のべき乗に反比例：

$$
N(x) = C x^{-\tau}
$$

両対数プロットで**直線**になる。**スケール不変性**を持つ：どんなスケールで見ても同じ形状。

#### べき乗則の現実例

| 現象 | 法則名 |
|------|--------|
| 地震のサイズ vs 頻度 | **Gutenberg-Richter 則** |
| 山火事の規模 | べき乗 |
| 金融市場の暴落 | べき乗 |
| 神経活動（ニューロン雪崩） | べき乗 |
| 種の絶滅 | べき乗 |
| 都市規模（人口） | **Zipf の法則** |
| 所得分布（高所得側） | **パレート分布** |

これらすべてが**同じ法則で記述される**のは、SOC が普遍的なメカニズムであることを示唆している。

### 「黒いスワン」とリスク管理

正規分布の前提だと、極端な事象は「**ほぼあり得ない**」と評価される。しかし**べき乗則の世界**では、

- 大規模イベントは稀だが、小規模と**同じ法則**で記述される
- 「**裾が太い**」分布になる

つまり「100年に1度の出来事」が**実は思ったより頻繁に起きる**。

リーマンショック、福島原発事故、コロナのような出来事は、**正規分布前提のリスク評価では過小評価**される。これが「**ブラック・スワン**」問題。べき乗則を前提にした **Tail Risk** 管理が必要。

### カオス・フラクタル・1/f ノイズ

複雑系に関連する概念：

- **カオス**：決定論的なルールから生じるが、**初期値に敏感**で予測困難な挙動。例：ローレンツ方程式、ロジスティック写像。「バタフライ効果」
- **フラクタル**：**部分が全体に似る**（自己相似）。マンデルブロ集合、海岸線、雲、樹木の枝分かれ
- **1/f ノイズ**：パワースペクトルが周波数に反比例するゆらぎ。心拍、音楽、太陽黒点

これらに共通するのが**スケール不変性**。SOC はそのメカニズム的説明を与える。

## 第12回：連続体の離散モデル化

### なぜ離散化するか

これまでの講義のモデルはすべて**離散**（マルコフ、CA、エージェント）。なぜか？

理由は2つ：

1. **コンピュータは離散計算しかできない**：浮動小数点も有限精度。シミュレーションは本質的に離散
2. **連続現象を扱うには PDE（偏微分方程式）を解く必要がある**：解析的に解けないので、**差分方程式**に置き換えて数値的に解く

### 連続体近似

ミクロ（原子・分子レベル）から見れば物質は粒子の集まり。でもマクロで見ると**連続的**な物体に見える。

| 巨視変数 | 微視からの意味 |
|----------|----------------|
| 密度 ρ | 微小体積内の粒子数 / 体積 |
| 速度 v | 粒子の平均速度 |
| 内部エネルギー | 平均運動エネルギー |

このように**集約**して扱うのが連続体力学。

### Navier-Stokes 方程式（流体の運動方程式）

$$
\frac{\partial \mathbf{v}}{\partial t} + (\mathbf{v}\cdot\nabla)\mathbf{v} = -\frac{1}{\rho}\nabla p + \frac{\mu}{\rho}\nabla^2 \mathbf{v} + \mathbf{F}
$$

- v：流速、p：圧力、ρ：密度、μ：粘性、F：外力

実は数学的に「**解が常に滑らかに存在するか**」がミレニアム懸賞問題のひとつで、**未解決**。それでも近似的に数値解は計算できる。

### 数値解法の流れ

```
第一原理（分子動力学）
    ↓ 連続体近似
PDE（Navier-Stokes など）
    ↓ 離散化
代数方程式
    ↓ 計算機求解
数値解
```

### 離散化手法

| 手法 | 概要 | 用途 |
|------|------|------|
| **有限差分法（FDM）** | 微分を**差分**で近似 | 単純な格子の流体解析 |
| **有限要素法（FEM）** | 要素分割 + **基底関数** | 構造解析、複雑形状 |
| **有限体積法（FVM）** | 体積積分の**保存則**を満たす | 流体（CFD）、保存則重視 |
| **粒子法（SPH）** | **ラグランジュ粒子**を追跡 | 自由表面、衝突 |

### 格子と時間刻みのトレードオフ

- **空間**：領域を格子（メッシュ）に分割。細かいほど精度↑、計算量↑
- **時間**：時間刻み Δt で前進。小さいほど精度↑、計算量↑
- **CFL 条件**：双曲型 PDE では時間刻み Δt は空間格子 Δx より十分小さく取る必要がある（さもないと数値が発散）

### 妥当性評価

モデルは**近似**にすぎない。本当に正しいかは：

- 実データ・実験との比較
- 境界条件・初期条件の妥当性
- スケール（格子解像度）の選択

を慎重に検証する必要がある。

### 講義全体の系譜

| 回 | テーマ | キーワード |
|----|--------|------------|
| 1-2 | 構造 | ISM, DEMATEL, 可到達 |
| 3 | 動的・因果 | SD, R/B, システム原型 |
| 4-5 | 曖昧・選択 | ファジィ, AHP |
| 6-8 | 確率 | マルコフ, πP=π, M/M/1 |
| 9-10 | 離散 | CA, エージェント |
| 11 | 複雑 | SOC, べき乗 |
| 12 | 連続→離散 | PDE, 格子, CFD |

このように「**静的な構造 → 動的な因果 → 確率 → 離散 → 複雑系 → 連続体**」と、扱う対象が広がっていく構成。

## 試験対策の戦略

### 試験形式

- 出席・レポート **60点** + 期末試験 **40点**
- 持ち込み**なし**
- 講義全体から出題

**戦略上の含意**：

- レポート提出 60 点を取り切れば、試験 20 点でも合計 80 点で A 評価圏内
- 試験は 40 点配点なので、**A 狙いでも 25〜30 点で十分**
- 計算問題（DEMATEL、定常分布）で確実に取れば合格は固い

### 優先順位

最優先（**計算系**、確実に得点したい）：

- DEMATEL の全計算（X → N → T → D, R, D−R, D+R）
- マルコフ定常分布の計算（πP=π を解く）
- 可到達行列の計算（M = A+I の冪）

高優先（**論述系**、書ければ大きく得点）：

- システムの定義と境界（存在論 vs 認識論）
- ISM と DEMATEL の使い分け
- R∩A=R の意味と階層抽出
- 因果ループの R/B 判定
- マルコフ性の定義と意味
- SOC とべき乗則
- 離散化の理由

中優先：

- ファジィと確率の違い
- AHP の手順と CR
- Game of Life や CA の規則
- 創発の例

### 論述の書き方

> **定義（何か） → なぜ重要か → 具体例 → 関連概念との繋がり**

の流れで書く。例：

「**マルコフ性**とは、確率過程において**次の状態が現在の状態だけで決まり、過去の履歴に依存しない**という性質である（定義）。この性質のおかげで、過去の全履歴を覚えなくても遷移確率だけで未来を予測できる（重要性）。例えば天気の遷移を考えるとき、今日が晴れなら明日の確率分布が決まり、昨日や一昨日の情報は不要（具体例）。これにより推移行列 P を用いた解析が可能になり、定常分布 πP=π によって長期挙動を計算できる（関連概念との繋がり）。」

### 試験当日のコツ

- **配点を見て時間配分**：60分なら、計算30分 + 論述25分 + 見直し5分
- **計算は途中式を残す**：行列計算、πP=π の連立、N と T の中間結果
- **論述は最初にアウトライン**：箇条書きで脱線防止
- **誤りの指摘問題**：「誤り。正しくは……」と明示
- **「3つ覚える」を活用**：システム3要素、確率公理3つ、分布関数の3性質など、まとめて思い出せるリスト
- **白紙提出は避ける**：分かるところだけでも書く

### 典型的な「誤りを指摘」型問題

| 誤った文 | 正しくは |
|----------|----------|
| 「DEMATEL は因果の正負を + と − で表す」 | DEMATEL は**非負の影響の強さ**（0〜4）を扱う。正負を扱うのは **Cognitive Map** |
| 「マルコフ連鎖では過去の全履歴が次の状態を決める」 | **現在の状態のみ**が次を決める（マルコフ性） |
| 「ISM は影響の強さを扱う」 | ISM は **0/1 関係のみ**。強さは DEMATEL |
| 「ρ ≥ 1 でも M/M/1 は安定」 | **発散**（到着 > 処理） |

### 典型的な「手法を選べ」型問題

| 状況 | 答え | 理由 |
|------|------|------|
| 階層が知りたい、関係は推移的 | **ISM** | 推移律と階層抽出 |
| 影響の強さ、原因/結果を知りたい | **DEMATEL** | 0〜4 評価、D−R |
| 正負の因果が本質 | **Cognitive Map** | 符号で表現 |
| 代替案からの選択 | **AHP** | 一対比較で重み |
| 時間変化を試算したい | **SD** | ストック/フロー |
| 状態遷移の不確実性 | **マルコフ** | 確率的状態遷移 |
| 待ち時間・サービス | **待ち行列** | M/M/1 |
| 局所ルールから大域へ | **CA / エージェント** | 創発 |
| 連続現象の数値解 | **PDE → 離散化** | FDM/FEM |

## 他科目との繋がり

このシステム工学基礎の枠組みは、他の科目を理解するための**メタ枠組み**として使える：

- **マネジメント原論**：社会的意思決定や PPP のフィードバック構造は、**システム思考・因果ループ**そのもの
- **応用物理 I**：熱力学のエントロピー増大則は、**SOC の臨界化現象**と類似の構造
- **建設マネジメント**：CPM の作業ネットワークは、**ISM の階層構造**そのもの
- **データ分析**：マルコフ・確率分布・モンテカルロは統計の基礎

つまり「**システム工学基礎は、他のすべての科目を解釈するための道具**」と位置づけると、各概念の意味が見えやすくなる。
