# 持ち込み用 — 論述テンプレ集

試験で出るパターン別に「**こう書けば点が来る**」テンプレを集めたもの。問題タイプを見たら該当テンプレを写経して、自分の問題に当てはめる。

## パターン 1：「○○と××の違いを述べよ」

**書き方**：表で対比 + 1〜2 文の補足

### テンプレ

```
○○と××の主な違いは以下の通り：

| | ○○ | ×× |
|---|----|----|
| 対象 | ... | ... |
| 性質 | ... | ... |
| 用途 | ... | ... |
| 例 | ... | ... |

両者は[補完関係 / 対立 / 別の不確実性 / etc.]にあり、
[使い分けの基準]に応じて選択する。
```

### 完成例 1：ISM vs DEMATEL

ISM と DEMATEL の主な違いは以下の通り：

| | ISM | DEMATEL |
|---|-----|---------|
| 関係の表現 | 0/1（あり/なし） | 0〜4（強さ） |
| 推移律 | 仮定する | 仮定しない |
| 入力データ | 一対比較 | アンケート・専門家評価 |
| 主目的 | 階層化 | 影響度・原因/結果の定量 |
| 出力 | 階層図 | D−R / D+R グラフ |

両者は補完関係にあり、関係が推移的で階層を知りたいときは ISM、影響の強さや原因系/結果系を区別したいときは DEMATEL を使う。実務では DEMATEL で重要要素を絞り込んでから ISM で階層化する併用もある。

### 完成例 2：確率 vs ファジィ

確率とファジィは異なる種類の不確実性を扱う：

| | 確率 | ファジィ |
|---|------|----------|
| 対象 | 事象の**発生**の不確実性 | 概念の**境界の曖昧さ** |
| 数学的基盤 | Kolmogorov の確率公理 | メンバーシップ関数 μ(x) ∈ [0,1] |
| 例 | 雨が降る確率 60% | 「高齢」の度合い 0.7 |
| 解釈 | 何度も繰り返したら 60% で起きる | 「高齢」概念に 70% 属する |

両者は対象が異なるため別の枠組みであり、現象に応じて使い分ける。確率はマルコフ連鎖や待ち行列、ファジィはエアコンの自動制御や FCM などに使われる。

### 完成例 3：ストック vs フロー

ストックとフローは SD の基本概念で、次のように対比される：

| | ストック | フロー |
|---|----------|--------|
| 意味 | 時点で測れる蓄積量（状態変数） | 単位時間あたりの増減量（変化率） |
| 数学 | 積分量 | 微分量 |
| 例 | 人口、貯金、CO₂濃度、在庫 | 出生率、入金、排出率、出荷率 |

ストックの時間微分 = 流入フロー − 流出フロー という関係が成り立つ。ストックは過去の蓄積を持つため、システムに**慣性・遅延**を生む。これが「税制を変えても人口がすぐ変わらない」理由。

### 完成例 4：R ループ vs B ループ

因果ループには 2 種類ある：

| | R（強化ループ） | B（バランスループ） |
|---|--------------|-------------------|
| 挙動 | 自己強化・発散 | 均衡・収束 |
| O 個数 | 偶数 | 奇数 |
| 例 | 貯金 → 利息 → 貯金 | 体温 → 発汗 → 体温 |

判定方法は、S を +1、O を −1 として全体の積を取る。積が +1 なら R、−1 なら B。同じことだが、O が偶数個なら R、奇数個なら B と覚える方が早い。R は自己強化の悪循環や複利的成長を生み、B は安定化やフィードバック制御の役割を果たす。

### 完成例 5：SM vs DM

構造モデリング（SM）と動的モデリング（DM）は補完的な 2 つのアプローチ：

| | SM | DM |
|---|-----|-----|
| 焦点 | 関係・階層（静的） | 時間変化・フィードバック（動的） |
| 代表手法 | ISM, DEMATEL, CM, AHP | SD（システムダイナミクス） |
| 問い | 「何が何に影響するか」 | 「時間とともにどう変わるか」 |
| 用途 | 整理・合意形成 | 挙動予測・政策実験 |

両者は補完関係にあり、まず SM で構造を整理してから DM で時間発展をシミュレーションするのが典型的な流れ。

## パターン 3：「○○の誤りを指摘せよ」

**書き方**：①誤り箇所を指摘 → ②正しくは何か → ③関連する正しい概念

### テンプレ

```
誤り。正しくは[正しい記述]である。
[なぜそうなるかの根拠 1〜2 文]。
[混同しやすい類似概念との区別]。
```

### 完成例 1：「DEMATEL は因果の正負を + と − で表す」

誤り。正しくは、DEMATEL は**非負の影響の強さ（0〜4）**を扱う手法であり、因果の正負（+/−）の符号は扱わない。

DEMATEL では各要素間の影響度を 0（なし）〜4（非常に強い）で評価し、直接影響行列 X、正規化行列 N、総合影響行列 T を計算する。出力の D−R は正負の値を取るが、これは「原因系か結果系か」の判別であって、要素間の因果の符号ではない。

因果の正負（+/−）を扱う手法は **Cognitive Map（CM）** や **FCM（Fuzzy Cognitive Map）** であり、これらは DEMATEL とは別の手法。

### 完成例 2：「マルコフ連鎖では過去の全履歴が次の状態を決める」

誤り。正しくは、マルコフ性により、**現在の状態のみ**が次の状態を決定し、過去の履歴は不要である。

式で書けば：

P(X_{n+1}=j | X_n=i, X_{n-1}, X_{n-2}, ...) = P(X_{n+1}=j | X_n=i)

つまり過去の履歴 X_{n-1}, X_{n-2}, ... はすべて忘れて良い（条件付き独立）。これにより推移確率行列 P だけで未来が予測でき、長時間後の挙動も πP = π の定常分布として解析できる。

過去の履歴を必要とするモデル（自己回帰 AR モデルなど）は、マルコフ連鎖ではない別の確率過程である。

### 完成例 3：「ISM は影響の強さを定量的に扱える」

誤り。正しくは、ISM は**0/1 の関係**のみを扱い、影響の**強さ**は扱えない。

ISM では一対比較で「i は j に影響するか（はい/いいえ）」を 0/1 で記録し、推移律を仮定して階層構造を抽出する。あくまで関係の**有無**のみを扱う。

影響の強さを 0〜4 で評価し、原因系/結果系を判別したい場合は **DEMATEL** を使う。両者は目的と前提が異なる別手法である。

### 完成例 4：「ρ ≥ 1 でも M/M/1 待ち行列は安定」

誤り。正しくは、M/M/1 では **ρ < 1 でのみ安定**で、ρ ≥ 1 では到着が処理を上回り、待ち行列が無限に伸びて発散する。

利用率 ρ = λ/μ は「サーバが忙しい時間の割合」を表し、ρ ≥ 1 は「サービス能力を超えて顧客が到着する」状態。この場合、平均行列長 L = ρ/(1−ρ) は無限大になり、系内の人数が時間とともに無限に増える。

実務では ρ → 1 に近づくと待ち時間が急増するため、稼働率は 80% 程度に抑えるのが目安となる。

## パターン 4：「○○な状況にはどの手法を使うか」

**書き方**：①状況の特徴を分解 → ②必要な出力 → ③手法選択 → ④理由

### テンプレ

```
[状況の特徴]という条件から、必要なのは[必要な出力]である。
したがって**[手法名]**が適している。
理由は、[その手法がその条件にどう合致するか]。
```

### 完成例（5 状況）

| 状況 | 適した手法 | 理由 |
|------|-----------|------|
| (a) SNS のフォロー関係、影響の強さを定量化したい | **DEMATEL** | フォロー関係は推移律が成り立たない（A が B をフォロー、B が C をフォローでも、A は C をフォローするとは限らない）ため、推移律を仮定する ISM は不適。DEMATEL なら 0〜4 の影響強度で要素間の関係を定量化できる。 |
| (b) 製品評価基準のもとで 3 案の優先順位を決めたい | **AHP** | 複数の評価基準下で代替案を順位付けする問題は AHP の典型用途。一対比較で重みを決め、各基準下のスコアを統合する。CR < 0.1 で判断の一貫性も検証できる。 |
| (c) 要因 A が要因 B を「増やす/減らす」の符号が本質 | **Cognitive Map** | 因果の正負（+/−）を扱える手法は Cognitive Map または FCM。DEMATEL は非負の強さのみ、ISM は 0/1 のみで符号は扱えない。 |
| (d) 影響が推移的、上位下位の階層が欲しい | **ISM** | 推移律と階層化が ISM の前提と目的に合致。可到達行列から R(i)∩A(i)=R(i) で最上位レベルを抽出し、階層図を作成する。 |
| (e) 政策介入後 10 年間の人口変化を試算したい | **SD（システムダイナミクス）** | 時間発展のシミュレーションが必要な動的問題。ストック（人口）とフロー（出生・死亡・転入・転出）を定義し、フィードバックループを組み込んでシミュレーションする。 |

## パターン 5：「○○の手順を示せ」

**書き方**：番号付きのステップで明確に列挙

### DEMATEL 手順

1. **直接影響行列 X の作成**：各要素間の影響を 0〜4 で評価
2. **正規化**：各行の行和を求め、最大行和で X を割って正規化行列 N を得る
3. **総合影響行列 T の計算**：T = N(I−N)⁻¹ = N + N² + N³ + ... により直接影響と間接影響をすべて合計
4. **D, R の計算**：D は T の各行の和（発信力）、R は各列の和（被影響力）
5. **D−R, D+R の計算**：D−R は因果性（正なら原因系、負なら結果系）、D+R は関連度（システム内での中心性）
6. **可視化**：D−R を縦軸、D+R を横軸に取った散布図（NRM）で要素を分類

### ISM 手順

1. **要素抽出**：分析対象となる要素を列挙
2. **一対比較**：「i は j に影響するか」を 0/1 で評価し、隣接行列 A を構築
3. **可到達行列の計算**：M = A + I の冪乗（M^(n−1) まで）を計算
4. **集合 R, A の作成**：各要素 i について R(i)（到達可能集合）と A(i)（先行集合）を可到達行列から読み取る
5. **レベル抽出**：R(i) ∩ A(i) = R(i) を満たす i を最上位レベルとして抽出。抽出した要素を除いて反復
6. **階層図の作成**：各レベルを縦に並べ、矢印で関係を示す

### マルコフ定常分布計算

1. **推移確率行列 P の確認**：各行の和 = 1 であることをチェック
2. **πP = π を成分ごとに書く**：n 個の方程式（実質 n − 1 個が独立）
3. **正規化条件 Σπ_i = 1 を加える**
4. **連立方程式を解く**：n 元 1 次連立を解いて π_i を求める
5. **検算**：得られた π について πP = π と Σπ_i = 1 を確認

### サンドパイル更新規則

1. **砂粒の追加**：ランダムに 1 セルを選び、値を +1
2. **崩壊判定**：値が 4 以上になったセルから 4 を減らし、上下左右の 4 近傍（von Neumann 近傍）に +1 ずつ加える
3. **連鎖崩壊の処理**：崩壊により他のセルも 4 以上になれば、すべて 3 以下に戻るまで 2 を繰り返す

安定したら次の砂粒へ。崩壊の規模はべき乗分布になり、これが自己組織化臨界（SOC）の典型例。

## パターン 6：「○○のメリット・デメリットを述べよ」

**書き方**：箇条書きで列挙 + 1〜2 文の補足

### テンプレ

```
**メリット**：
- ...
- ...

**デメリット・限界**：
- ...
- ...

総合的に、[使い分けの結論]。
```

### 完成例：DEMATEL のメリット・デメリット

**メリット**：

- **影響の強さを定量化**できる（0〜4 で評価）
- **間接影響も含めて集計**できる（T = N + N² + ... で連鎖を捕捉）
- **原因系/結果系を判別**できる（D−R で分類）
- **推移律を仮定しない**ので、フォロー関係や相性のような関係にも適用可能

**デメリット・限界**：

- **評価が主観的**になりがち（0〜4 の判定が人によって違う）
- **専門家の評価が必要**で、多人数のアンケートが負担
- **間接影響を含めるため、計算結果の解釈が複雑**になる
- **時間軸を扱えない**（動的挙動は SD など他手法で）

総合的に、要因の影響度を定量化したい構造分析に最適だが、定量的な精度を求めるなら統計的手法、時間発展を見たいなら SD を併用する。

## パターン 7：「事例分析・応用」

**書き方**：①事実の概要 → ②構造的要因 → ③用語による分析 → ④結論

### テンプレ

```
[事例の概要を 1〜2 文で]

この事例を[該当する手法・概念]で分析すると：
- [要因 1]
- [要因 2]
- [要因 3]

これは[システム原型・概念名]の典型例で、
[結論または教訓]。
```

### 完成例：マニラ水道の悪循環

マニラ水道では「低料金 → 収入不足 → メンテ投資不足 → 漏水・サービス悪化 → 解約・収入減」という悪循環が観察された。

この事例を因果ループ図で分析すると、すべての連鎖が S（同方向変化）で構成された強化ループ R になっている：

- 料金↓ →(S) 収入↓
- 収入↓ →(S) メンテ投資↓
- メンテ投資↓ →(O) 漏水↑（投資が減れば漏水が増える）
- 漏水↑ →(S) サービス悪化↑
- サービス悪化↑ →(S) 解約↑
- 解約↑ →(S) 収入↓（戻る）

O が複数個ある複雑なループだが、全体の積は + で R（自己強化）。これは Senge の**システム原型「Fix That Fails」**の典型例で、低料金という応急処置が長期的な悪化を生んでいる。脱出には適切な料金体系の設計と段階的な投資が必要。

## パターン 8：「(1)〜(5) の小問形式」

**書き方**：各小問に最小限の答え + 必要なら 1 文の理由

### 完成例（基本問題形式）

**(1) システムを構成する 3 つの要素を挙げよ**

要素・相互作用・目的性。

**(2) 良定義問題と悪定義問題の違いを述べよ**

良定義問題は目標・手段・評価基準が明確で最適化で解ける問題。悪定義問題（wicked problem）は目的・関係者・解が曖昧で合意形成が困難な問題。前者は数学的最適化、後者は構造化手法やソフトシステムズ方法論が有効。

**(3) ISM の手順を 5 ステップで示せ**

①要素抽出 → ②一対比較で隣接行列 A 作成 → ③M = A + I の冪乗で可到達行列 → ④R(i) ∩ A(i) = R(i) でレベル抽出 → ⑤階層図作成。

**(4) M/M/1 の利用率 ρ の定義と、ρ ≥ 1 のときの挙動を述べよ**

ρ = λ/μ（到着率÷サービス率）。ρ ≥ 1 では到着が処理を上回り、待ち行列が無限に伸びて発散する（不安定）。

**(5) SOC を 1 文で説明せよ**

外部調整なしに系が自分で臨界状態に向かい、それを保つ現象で、サンドパイル・モデルにおける崩壊規模のべき乗分布が典型例。

## 試験中の論述の鉄則

1. **問題タイプを最初に判断**：違い説明、誤り指摘、用語説明、手順、事例、適切手法選択
2. **該当テンプレを開いて写経**：問題の用語に置き換える
3. **箇条書きや表で書ける部分は表で**：採点者に伝わりやすい
4. **具体例を 1 つは入れる**：定義だけより点が伸びる
5. **「白紙にしない」**：分からなくても、関係しそうなキーワードを書いておく
6. **時間配分を守る**：1 問に 15 分以上かけない
