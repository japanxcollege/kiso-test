# 2015 年度（平成 27 年度）期末試験

## 第 1 問

### 1-1：可到達行列・総合影響行列から得られる知見

**可到達行列（表 1）**（ISM 法）：
- 行列の (i, j) 要素が 1 なら、デザイン項目 i から項目 j に到達可能（影響あり）
- 各行の合計が大きい行は **発信力が大きい要素**（多くの項目に影響を与える）
- 各列の合計が大きい列は **被影響力が大きい要素**（多くの項目から影響を受ける）
- これにより、**システムの階層構造**（どの要素が上位の根本要因か）が判別できる
- 階層抽出条件 R(i) ∩ A(i) = R(i) を満たす要素が最上位レベル

**総合影響行列（表 2）**（DEMATEL 法）：
- 値が **影響の強さの定量値**を示す
- **D（行和）= 発信力**：その要素が他に与える総影響量
- **R（列和）= 被影響力**：その要素が他から受ける総影響量
- **D − R > 0 → 原因系**（介入すべき根本要素）
- **D − R < 0 → 結果系**（症状にあたる要素）
- **D + R = 関連度**（システム内での中心性）

**情報処理方法**：
1. 可到達行列で関係の **有無** を判定 → 階層化
2. 総合影響行列で関係の **強さ** を定量化 → 原因系/結果系を分離
3. 両者を組み合わせて、**重要な介入ポイントの特定** や **設計上の優先順位付け** が可能

### 1-2：システムズ・アプローチの用語論述

> □システム□ とは、<u>複数の要素が相互に関係を持ち、全体として一つの目的を達成する集合体</u>である。**=システムの認識=** において、まず □システム境界□ を引くことが重要で、<u>=境界の引き方は分析者の観点で変わる=</u>。
>
> □システム概念□ では、<u>要素・相互関係・目的性の 3 要素を持つ全体として捉える</u>。<u>=□要素□ はシステムを構成する個々の実体（人・物・情報）であり=</u>、<u>=□相互関係□ はそれらの間の影響・因果・流れである=</u>。
>
> これらを統合的に扱う思考法が □システム思考□ である。<u>線形思考と異なり、要素間のフィードバックや相互依存を考慮する</u>。<u>=システム思考により、部分最適に陥らず全体最適を目指す設計が可能になる=</u>。

### 2-1：平均頂点間距離（無向グラフ）

各頂点ペア (i, j) の最短経路長を計算し、それらの平均：

$$
\bar{d} = \frac{\sum_{i < j} d(i, j)}{\binom{n}{2}}
$$

8 頂点なら 28 通りのペアの距離を求めて平均する（具体値は隣接行列の構造に依存）。

### 2-2：A 社の携帯電話を持つ確率（マルコフ連鎖）

A 社の携帯ユーザのうち：
- 0.2 の確率で買い替え（うち 0.8 で B 社、0.2 で A 社）
- 0.8 の確率で買い替えない（A 社のまま）

→ A 社 → A 社の遷移確率：

$$
P(A \to A) = 0.8 + 0.2 \times 0.2 = 0.84
$$

B 社のユーザのうち：
- 0.3 の確率で買い替え（うち 0.7 で A 社、0.3 で B 社）
- 0.7 の確率で買い替えない

→ B 社 → A 社の遷移確率：

$$
P(B \to A) = 0.3 \times 0.7 = 0.21
$$

#### 2-2 (2-1)：2 年目の確率

初め A 社（確率 1）→ 2 年目：

$$
P_2(A) = P(A \to A) \times 1 + P(B \to A) \times 0 = 0.84
$$

ただし「**2 年目**」が 1 ステップ後なら 0.84。2 ステップ後なら：

$$
P_2(A) = 0.84 \times 0.84 + 0.16 \times 0.21 \approx 0.7392
$$

#### 2-2 (2-2)：n 年目の確率

推移行列：

$$
P = \begin{pmatrix} 0.84 & 0.16 \\ 0.21 & 0.79 \end{pmatrix}
$$

定常分布 π = (π_A, π_B) は πP = π かつ π_A + π_B = 1：

- 0.84 π_A + 0.21 π_B = π_A → 0.21 π_B = 0.16 π_A → π_B = (16/21) π_A
- π_A + (16/21) π_A = 1 → (37/21) π_A = 1 → π_A = 21/37 ≈ 0.568

n → ∞ で **π_A ≈ 0.568**（21/37）。

n 年目の正確な確率は P^n の (1,1) 成分。

# 2016 年度（平成 28 年度）期末試験

## 第 1 問

### 1-1：システムズ・アプローチの用語論述

> □システム概念□ とは、<u>複数の □システム要素□ が □相互関係□ を持ち、全体として目的を達成するもの</u>である。**=□システムの認識□ では、何をシステムとし、何を境界の外とするかを決定することが分析の出発点となる=**。
>
> □システム思考□ は、<u>=要素間のフィードバック・相互依存を考慮する全体論的な思考法=</u> で、線形思考の対極にある。

### 1-2：構造化手法の用語論述

> □構造化手法□ とは、<u>要素間の関係を行列で表現し、システムの構造を可視化する手法群</u>である。ISM 法では、<u>=□一対比較□ を用いて要素間の関係を 0/1 の □バイナリ行列□ として □隣接行列□ に表す=</u>。
>
> <u>=□推移律□ を仮定し=</u>、<u>=□ブール代数演算則□ に従って M = A + I の冪を計算する=</u> ことで、ある回数で **収束** し、=□可到達行列□ が得られる=。<u>=冗長な推移関係を除いた □骨格行列（スケルトン行列）□ で階層構造を可視化する=</u>。

### 1-3：マン・マシン系の図の説明

マン・マシン系は **人間** と **機械** が **インターフェース** を介して **相互作用** するシステム。

要素：
- **人間**：認識・判断・意思決定を行う主体
- **機械**：人間の操作に応じて動作する物理系（例：自動車、PC）
- **インターフェース**：両者をつなぐ媒介（ハンドル、画面、キーボード等）
- **環境**：人間・機械が置かれている状況
- **タスク**：人間と機械が共同で達成する目標

### 1-4：DEMATEL 法の用語論述

> □構造化手法□ の一つである DEMATEL 法では、<u>=□一対比較□ により要素間の影響度を 0〜4 で評価し、□直接影響行列□ を作成する=</u>。次に <u>=正規化と冪和 N + N² + N³ + ... で □間接的影響行列□ を求める=</u>。
>
> 両者を合わせた総合影響行列 T から、<u>=各要素の発信力 D（行和）と被影響力 R（列和）を計算し、□影響度（D − R）□ で原因系/結果系の判別、□関連度（D + R）□ でシステム内の中心性を評価する=</u>。

### 1-5：Cognitive Map（CM）

Cognitive Map（CM）は、**要素間の因果関係を符号付き有向グラフ**で表現する手法。

- **+** リンク：原因が増えれば結果も増える（正の因果）
- **−** リンク：原因が増えれば結果が減る（負の因果）

ISM や DEMATEL と異なり、**因果の符号（増減方向）**を表現できる。Axelrod（1976）が政治学・経営学で使用したのが起源。

### 2-2：非決定性 → 決定性オートマトンの変換

部分集合構成法で変換：

1. 初期状態：{q_0}（NFA の初期状態を含む単集合）
2. δ'(S, a) = ∪_{q ∈ S} δ(q, a) を計算
3. 新状態が出たら追加
4. 受理状態：F' = {S | S ∩ F ≠ ∅}

具体例（4 状態 NFA を 2 入力で決定化）：

| Q' | 元 NFA の状態 | 入力 a | 入力 b |
|----|--------------|-------|-------|
| q_0' | {q_0} | {q_1} = q_1' | ∅ |
| q_1' | {q_1} | {q_2} = q_2' | {q_3} = q_3' |
| q_2' | {q_2} | ∅ | {q_2} |
| q_3' | {q_3} | {q_1, q_3} = q_4' | ∅ |
| q_4' | {q_1, q_3} | {q_1, q_2, q_3} | {q_3} |
| ... | ... | ... | ... |

実際の値は元の NFA の遷移関数に依存。

### 3-2：エージェントとマルチエージェントシステム

**(a) 単純条件反射エージェント vs モデルベース条件反射エージェント**

先輩解答より：

> □単純条件反射エージェント□ は、<u>現在の状態に適合するルールを見つけて、そのルールに合う動作を行うもので、完全観測可能な環境のみでしか適用できない</u>。
> 
> それに対し、□モデルベース条件反射エージェント□ は、<u>内部状態を持ち、時間経過とともにそれを更新することで、単純反射エージェントの適用範囲を、完全観測可能でないものにまで広げている</u>。

**(b) 静的タスク環境**

> エージェントの行動の影響は **その場限りで、未来に影響しない** 環境のこと。

例：チェスや将棋のように、エージェントが行動を選ぶ間に環境が変わらない環境。

**(c) 競争的マルチエージェント環境**

複数のエージェントが**互いの利得を最大化しようとして対立する**環境。一方の利得が他方の損失になる（ゼロサム）場合が多い。

例：オークション、対戦ゲーム、市場での競合企業。

## 第 2 問（必答・50 点）

### (1)(a) 反応性 と 自発性・先見性

> **反応性**：エージェントが環境の変化を**知覚し、即座に反応**する能力。事前に決められたルールに従って動く。
> 
> **自発性・先見性**：エージェントが**自分の目標に向かって主体的に行動**する能力。環境の変化を待つだけでなく、未来を予測して計画的に動く。

### (1)(b) スケール不変（スケールフリー）

> 複雑系で、**べき乗則 N(x) ∝ x^(−τ) に従い、特定のスケールに依存しない性質**。両対数プロットで直線になる。地震のサイズ vs 頻度（Gutenberg-Richter 則）、所得分布（パレート分布）、都市規模（Zipf 則）など。

### (1)(c) 平均場近似

> 多数の要素が相互作用する系で、**他の要素の影響を「平均値」で代表**させて各要素を独立に扱う近似手法。複雑な多体問題を扱いやすくする。例：イジングモデルでスピン間の相互作用を平均磁場で近似する。

## 第 3 問（選択・50 点）：因果ループ・SD・ファジィ

### (1) 因果ループ図の説明

「商品の魅力」「売上金額」「商品開発の原資」の 3 要素の関係：

- 商品の魅力↑ → 売上金額↑（S：同方向）
- 売上金額↑ → 商品開発の原資↑（S：同方向）
- 商品開発の原資↑ → 商品の魅力↑（S：同方向）

これは **O が 0 個（偶数）の強化ループ R**。商品の魅力が高まれば売上が増え、開発原資が増え、さらに魅力が高まる **自己強化の好循環**。逆向きに動けば自己強化の悪循環にもなる。

### (2) ストック・フロー・コンバータ

- **ストック**：「**商品開発の原資**」（累積する資金の量、状態変数）、または「**商品の魅力**」も累積効果あり
- **フロー**：「**売上金額**」（時間あたりに発生する収入）、「**開発投資**」（時間あたりに使う金額）
- **コンバータ**：「商品の魅力 → 売上金額」の変換係数、「売上金額 → 開発原資」の比率など

理由：
- ストックは**時点で測れる蓄積量**（過去の累積）
- フローは**単位時間あたりの増減量**（変化率）
- コンバータは**フローやストックを計算する中間値**

### (3) ファジィ概念・集合・数・関係

- **ファジィ概念**：「重い」「軽い」「高い」のように**境界が曖昧な概念**
- **ファジィ集合**：要素の所属を [0, 1] の度合いで表す集合。μ_A(x) ∈ [0, 1]
- **ファジィ数**：「だいたい 5」「約 10」のような数を表すファジィ集合
- **ファジィ関係**：要素間の関係をファジィ集合で表したもの（「A は B にやや似ている」など）

### (4) 「重い」「軽い」のメンバーシップ関数

```
μ
1 ─┬─────┐ μ_軽い
   │     \\
   │      \\
   │       \\
   │        \\        μ_重い 
   │         \\─────┌───
   │          \\   /
   │           \\ /
   │            X
   │           / \\
   │          /   \\
0 ─┴─────────/─────\\──── 重量 (kg)

```

- μ_重い：[0, 2] で 0、[2, 5] で線形に 1 へ、[5, ∞] で 1

>

### (4) 2 次元ランダムウォーク：2n 回目に原点に戻る確率

xy 平面上の原点から出発、各方向 1/4 で移動。

2 次元ランダムウォークは独立な 1 次元 ×2 と見なせる。x 方向と y 方向の動きの配分を考えると：

$$
P_{2n}(0,0) = \binom{2n}{n}^2 \left(\frac{1}{4}\right)^{2n}
$$

スターリングの公式 $n! \approx \sqrt{2\pi n} (n/e)^n$ より：

$$
\binom{2n}{n} \approx \frac{4^n}{\sqrt{\pi n}}
$$

よって：

$$
P_{2n}(0,0) \approx \left(\frac{4^n}{\sqrt{\pi n}}\right)^2 \cdot \frac{1}{16^n} = \boxed{\frac{1}{\pi n}}
$$

→ 2 次元ランダムウォークは**再帰的**（n→∞ で確率の総和は無限大になる）が、確率自体は **1/(πn)** で減衰する。

> - 「もし 需要逼迫 が大 かつ 供給余裕 が小 ならば → 蓄電池を放電 する」
> - 「もし 価格 が高い かつ ユーザー反応 が敏感 ならば → 需要応答プログラムを発動」
>
> これにより、**=不確実で曖昧な状況下でも合理的な自動制御判断= ができる**。エアコンの自動運転や新幹線の自動運転制御と同じ原理。

## 第 2 問（必答）

### (1) フラクタル次元

**(ア) 「カ」の字**：図形から N と s を読み取る。例：N = 4、s = 3 なら D = log 4 / log 3 ≈ 1.262

**(イ) 「ヨ」の字**：同様に図形から読み取る。例：N = 5、s = 3 なら D = log 5 / log 3 ≈ 1.465

**(ウ) ドラゴン曲線**：手順 1〜3 から自己相似性を読み取る。典型的なドラゴン曲線なら D = log 2 / log √2 = 2、または D = log 4 / log(2√2) = 4/3 など。

実際の値は問題図に依存。**全体を s 倍に拡大すると N 個のコピーで構成される** という関係を読み取って公式に代入。

### (2) 非決定性 → 決定性オートマトン

問題図の M(Q, Σ, δ, q_0, F)：4 状態 r_0, r_1, r_2, r_3、入力 0, 1。

部分集合構成法（前述）で変換。

### (3) 各項目の説明

**(ア) 合理性 と 完全性**

> **合理性**：エージェントが**与えられた情報と目標に基づいて、効用を最大化する選択をする**性質。
>
> **完全性**：エージェントが**環境のすべての情報を持ち、すべての選択肢を考慮できる**性質。
>
> 違い：合理性は「**与えられた情報の中で最適か**」、完全性は「**すべての情報を持っているか**」。チェス AI は計算範囲内で合理的だが、すべての手を読み切れないため完全ではない（**限定合理性**）。

**(イ) 単純条件反射エージェント と モデルベース条件反射エージェント**

> **単純条件反射エージェント**：現在の状態（知覚）に適合する IF-THEN ルールを見つけて、そのルールに合う動作を行う。**完全観測可能な環境のみで適用可能**。例：サーモスタット。
>
> **モデルベース条件反射エージェント**：**内部に世界モデルを持ち**、時間経過とともにそれを更新する。これにより、**見えない部分を推測して**行動できる。**部分観測可能な環境にも適用可能**。例：自動運転車。

**(ウ) スケール不変（スケールフリー）**

> 複雑系の特性で、**べき乗則 N(x) ∝ x^(−τ) に従い、特定のスケールに依存しない性質**。両対数プロットで直線になる。地震のサイズ vs 頻度（Gutenberg-Richter 則）、所得分布（パレート分布）、都市規模（Zipf 則）など。**正規分布前提では稀な極端事象が、思ったより頻繁に起きる**（ブラック・スワン）。

**(エ) 平均場近似**

> 多数の要素が相互作用する系で、**他の要素の影響を「平均値」で代表**させて各要素を独立に扱う近似手法。複雑な多体問題を扱いやすくする。例：イジングモデルでスピン間の相互作用を平均磁場で近似する。

# 試験の心得（過去問から見えること）

1. **必答 1 は用語論述**：問題文の用語を **□で囲む**、内容を **下線**、関連性を **二重線** で示す。配点ルール（1 点/内容、2 点/関連性）を必ず守る。

2. **フラクタル次元は公式が固定**：D = log N / log s。図形から N と s を正しく読み取る。

3. **オートマトンは部分集合構成法**：手順を覚えれば確実に取れる。

4. **エージェント特性は 4 用語セット**：反応性・自発性、合理性・完全性、単純反射・モデルベース、スケール不変・平均場近似。

5. **マルコフ・待ち行列は応用問題**：M/M/1 公式、走行割合の定常分布。

6. **同じ問題が複数年出題されている**：特に 2022 と 2023 は第 2 問が酷似。過去問解いておけば対応できる。

7. **2025 は応用化**：スマートシティのような題材に対し「**どう活用するか**」を問う形式。手法の本質を理解する必要あり。
